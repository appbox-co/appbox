# تقديم n8n على Appbox

أصبح n8n متاحاً الآن على Appbox.

هذا يعني أنه يمكنك تشغيل مساحة عمل خاصة لأتمتة سير العمل، وفتح المحرر المرئي، والبدء في ربط التطبيقات وواجهات API والجداول الزمنية وwebhooks والكود المخصص من دون إدارة خادم يدوياً.

## ما الذي يمكنك بناؤه

يكون n8n مفيداً كلما احتاج تطبيق إلى التفاعل مع تطبيق آخر، أو تنظيف البيانات، أو استدعاء API، أو إرسال إشعار، أو تشغيل مهمة مجدولة.

على Appbox، يجعله ذلك رفيقاً طبيعياً للتطبيقات التي توفر webhooks أو واجهات API أو notification hooks أو WebDAV أو RPC أو واجهات قابلة للبرمجة.

لسير عمل AI والدعم والأعمال:

- يمكن لـ **OpenClaw** أن يعمل كطبقة AI لأتمتتك. أرسل webhook من n8n عندما تحتاج تذكرة دعم أو تنبيه مراقبة أو رفع ملف إلى تلخيص، ثم دع OpenClaw يصوغ الرد أو يصنف الطلب أو يحول الحدث إلى مهمة.
- يوفّر **Chatwoot** أحداث inbox وواجهات API لدعم العملاء. ضع وسوماً على المحادثات العاجلة، أو أثْرِ جهات الاتصال الجديدة من نظام آخر، أو أبلغ قناة الفريق عندما يكتب عميل VIP، أو أنشئ مهمة متابعة بعد إغلاق محادثة.
- لدى **Mautic** واجهة REST API ومساحة لأتمتة الحملات. زامن leads من النماذج، وسجّل نقاط contacts بعد أحداث المنتج، وأضف الأشخاص إلى الحملات من إرساليات webhook، أو أبلغ فريق المبيعات عندما يتجاوز lead حداً معيناً.
- يتيح **ERPNext** سجلات الأعمال عبر واجهات API. أنشئ فواتير من إرساليات النماذج، وحدّث الصفقات عند وصول دفعة، وأبلغ العمليات عندما ينخفض المخزون، أو ادفع سجلات العملاء إلى تطبيق Appbox آخر.
- يعمل **Plane** و **OpenProject** جيداً كوجهات للمهام. حوّل الحوادث وإرساليات النماذج والمهام الفاشلة وطلبات العملاء إلى issues وwork packages مع المسؤول والسياق المناسبين.
- يمكن أن يناسب **Rallly** سير عمل الجدولة. أنشئ polls من نماذج الطلبات، وأبلغ الضيوف بعد اختيار موعد اجتماع، أو أرسل تذكيرات قبل الموعد النهائي لتحديد التوفر.
- يناسب **OnlyOffice** و **Jitsi** سير عمل المستندات والاجتماعات. أبلغ المراجعين عندما يصبح مستند جاهزاً، أو أنشئ رابط اجتماع بعد الموافقة، أو أرسل مهام المتابعة بعد الاجتماع إلى Plane أو OpenProject.

لسير عمل المحتوى والنشر والتطوير:

- يمنحك **EmDash** نظام CMS منظماً مع نماذج وروابط قصيرة وplugins وautobuild hooks. أرسل إرساليات النماذج إلى n8n، وأبلغ المحررين عند نشر المحتوى، وشغّل rebuild بعد انتقال مسودة إلى live، أو أرسل المنشورات الجديدة إلى Mattermost وRocket.Chat وقوائم الشبكات الاجتماعية.
- لدى **WordPress** واجهة REST API ونظام hooks واسع. أنشئ مسودات منشورات من إرساليات النماذج، وانشر إعلانات عندما يصبح المحتوى live، وانسخ الوسائط احتياطياً إلى تطبيق تخزين آخر، أو أرسل تذكيرات تحريرية قبل المنشورات المجدولة.
- يوفّر **GitLab** و **Gogs** وتطبيقات Git الأخرى hooks للمستودعات والمشاريع والنظام. شغّل إشعارات الإصدارات على tags، وأنشئ قوائم تحقق للنشر بعد merges، وانشر ملخصات pipelines الفاشلة في الدردشة، أو افتح issues لإدارة المشاريع من قوالب bug reports.
- يمكن أن يكون **Hugo Git** جزءاً من تدفق نشر قائم على Git. شغّل إشعارات rebuild بعد commits المحتوى، وانشر changelog عند وصول tag جديد، أو اطلب من n8n إنشاء مهمة مراجعة لكل pull request.
- يفيد **Docmost** كوجهة للتوثيق. أنشئ قائمة onboarding عندما يسجل عميل جديد، وانشر ملاحظات incident من أحداث المراقبة، أو أبقِ runbooks الداخلية محدثة من مخرجات سير العمل.
- يمكن أن يدعم **Jupyter Notebook** سير عمل البيانات والتقارير. شغّل notebooks مجدولة، وصدّر تقارير بعد تحديث قاعدة بيانات، أو أبلغ قناة عندما ينتهي تحليل طويل.
- يمكن أن يغذي **Serposcope** سير عمل SEO. أرسل تنبيهات لتغيّر الترتيب، وأنشئ ملخصاً أسبوعياً لمواضع البحث، أو افتح مهمة محتوى عندما تنخفض كلمة مفتاحية مهمة.
- تفيد **MySQL/MariaDB** و **Nginx & PHP** و **LAMP Stack** في الأدوات الداخلية. استخدم n8n لكتابة rows، أو استدعاء admin endpoints خاصة، أو تشغيل فحوصات صيانة مجدولة، أو إبلاغ فريق عندما يتلقى تطبيق مخصص إرسالية جديدة.

للملفات والتخزين والمزامنة والنسخ الاحتياطي:

- يوفّر **Nextcloud** WebDAV ومنظومة تطبيقات واسعة. انقل الملفات المرفوعة إلى مجلدات مراجعة، وأبلغ فريقاً عند وصول رفع من عميل، وأنشئ متابعات تقويم، أو انسخ المستندات المعتمدة إلى مساحات المشاريع.
- يتضمن **SFTPGo** واجهة REST API وإجراءات أحداث وlogin hooks. شغّل سير عمل لمعالجة الملفات عندما يرفع عميل تقريراً، وانقل الصادرات المكتملة إلى Nextcloud، وأبلغ قناة عند وصول نقل من شريك، أو عطّل المستخدمين المؤقتين بعد موعد نهائي.
- يوفّر **File Browser Quantum** و **File Browser WebDAV** و **Kodbox** مساحات لإدارة الملفات مثل REST APIs وWebDAV وcommand hooks أو storage plugins. راقب المجلدات المشتركة، وأنشئ metadata، وانسخ الأصول المنتهية إلى تخزين أرشيفي، أو وجّه الملفات للموافقة قبل النشر.
- يمكن برمجة **Rclone** عبر object storage وSFTP وWebDAV وcloud remotes. جدوِل مهام مزامنة من n8n، وتحقق من النقل بعد عمليات migration كبيرة، وانقل الوسائط المعالجة إلى تخزين طويل الأمد، أو أرسل ملخصات الفشل إلى الدردشة.
- يمنح **Restic REST Server** عملاء restic هدف نسخ احتياطي عبر HTTPS. أبلغ المسؤولين عندما تفشل فحوصات النسخ الاحتياطي المجدولة، وافتح incident بعد snapshots مفقودة، أو اكتب ملخصاً يومياً لصحة النسخ الاحتياطي في Docmost.
- يفيد **Syncthing** و **Resilio Sync** في خطوط أحداث الملفات. تفاعل عندما ينتهي مجلد من المزامنة، أو انسخ drops المكتملة إلى مكتبة وسائط، أو نبّه مستخدماً عندما لا تصل الملفات المتوقعة.
- يمكن أن تكون **Pure-FTPd** ونقاط FTP/SFTP الأخرى جزءاً من سير عمل ملفات قديم. افحص دورياً ملفات الشركاء الجديدة، وانقلها إلى SFTPGo أو Nextcloud، ثم أبلغ الفريق المناسب عند اكتمال المعالجة.

للأمان والوصول والبنية التحتية:

- يوفّر **Vaultwarden** واجهة API متوافقة مع Bitwarden وإشعارات SMTP اختيارية. أنشئ تذكيرات لمراجعات vault، أو أبلغ المسؤولين عند استحقاق مهام onboarding أو offboarding، أو نسّق قوائم تحقق لتدوير كلمات المرور بين الفرق.
- يمكن أن يكون **OpenVPN** و **WireGuard VPN** جزءاً من سير عمل الوصول. أبلغ المشغلين عندما تحتاج credentials إلى تدوير، وأنشئ مهام قائمة تحقق لمستخدمي VPN الجدد، أو شغّل فحوصات اتصال مجدولة عبر n8n.
- تمنحك تطبيقات **Ubuntu** و **Debian** و **Fedora** و **Rocky Linux** و **AlmaLinux** و **Arch Linux** و **Windows VPS** بيئات قابلة للبرمجة. استخدم n8n لتشغيل أوامر SSH، أو إجراء فحوصات صحة، أو جمع ملفات تقارير، أو نشر ملخصات صيانة بعد المهام المجدولة.
- يمكن أن تدعم تطبيقات **Ubuntu** و **Debian** و **Firefox remote desktop** مهام المتصفح أو سطح المكتب. جدوِل تذكيرات لأعمال المراجعة اليدوية، والتقط مخرجات scripts، أو سلّم صادرات سطح المكتب المكتملة إلى تخزين الملفات.

للمراقبة والدردشة والإشعارات:

- يدعم **Uptime Kuma** أهداف إشعارات وwebhooks غنية. حوّل أحداث التوقف إلى تنبيهات Mattermost أو Rocket.Chat، وأنشئ incidents في OpenProject، واطلب من OpenClaw ملخص incident قصير، أو سجّل كل تعافٍ في جدول حالة.
- يفيد **Mattermost** و **Rocket.Chat** كنقاط إشعارات وموافقات. أرسل تحديثات قابلة للقراءة من أي سير عمل، واطلب من فريق الموافقة على طلب، ثم واصل الأتمتة بعد أن يرد أحدهم.
- يفيد **RocketChat Hubot** في أوامر الدردشة. دع الزملاء يشغّلون webhooks في n8n من الدردشة، أو يطلبون حالة النشر، أو يبدأون سير عمل صيانة scripted مع audit trail.
- يصل **Matterbridge** بين IRC وMatrix وXMPP وSlack وDiscord وMicrosoft Teams وTelegram وغيرها. استخدم n8n لتنسيق الأحداث التشغيلية مرة واحدة، ثم ترحيلها إلى المجتمعات أو الغرف التي تحتاج إلى رؤيتها.
- يمكن أن يكون **ZNC** و **The Lounge** جزءاً من سير عمل chat-ops. مرّر أحداث IRC المهمة إلى قنوات فريق حديثة، وأرشف highlights، أو أبلغ المشغلين عند ظهور كلمة مفتاحية مراقبة.

لأتمتة الوسائط والتنزيلات:

- يوفّر **Tautulli** نشاط Plex وnotification scripts وJSON API. أرسل ملخصات الوسائط المضافة حديثاً، ونبّه عند حدوث خطأ في stream، وصدّر تقارير الاستخدام، أو أنشئ مهام تنظيف عندما يتجاوز التخزين حداً معيناً.
- يمكن أن تغذي **Plex** و **Jellyfin** و **Emby** و **Navidrome** و **Audiobookshelf** و **Calibre** و **Madsonic** و **SeAT** سير عمل مكتبات أو مجتمعات. أعلن عن وسائط جديدة، وحدّث المكتبات بعد imports، وأنشئ ملخصات استماع أو قراءة، وزامن metadata إلى dashboard، أو أبلغ مجتمعاً عندما يتغير نشاط متتبع.
- يمكن أن يناسب **Immich** سير عمل الصور. أبلغ العائلة عندما يتغير ألبوم مشترك، وانسخ الألبومات المصدرة إلى تخزين أرشيفي، أو شغّل فحص نسخ احتياطي بعد رفع كبير من الهاتف.
- يتضمن **AzuraCast** واجهة REST API وwebhooks. أعلن عن تغييرات tracks أو live shows، وحدّث widget في موقع ويب، وأبلغ المضيفين قبل بث مجدول، أو أرشف metadata قائمة التشغيل بعد برنامج.
- يمكن أن يدعم **Icecast** سير عمل حالة البث. راقب توفر stream، وانشر تحديثات now-playing، أو نبّه قناة عندما ينقطع source.
- يوفّر **Flood** و **ruTorrent** و **qBittorrent** و **Transmission** و **Deluge** واجهات RPC أو web APIs. راقب التنزيلات المكتملة، وانقل الملفات المنتهية إلى المكتبة الصحيحة، وشغّل scans للوسائط، أو أبلغ المستخدمين عندما يصبح عنصر مطلوب جاهزاً.
- يمكن أن تناسب **PufferPanel** و **TeamSpeak 3** وخوادم الألعاب أو الدردشة سير عمل تشغيلية أيضاً. أبلغ المسؤولين عندما يحتاج خادم إلى اهتمام، أو أنشئ تذكيرات صيانة، أو انشر إعلانات موجهة للاعبين بعد تغييرات مجدولة.

يمكنك أيضاً استخدام n8n مع خدمات خارجية عبر nodes مدمجة وwebhooks وطلبات HTTP واتصالات قواعد البيانات وcode nodes.

## تثبيت n8n

افتح [صفحة n8n في App Store](/appstore/app/264)، واختر Appbox الخاص بك، وابدأ التثبيت.

أثناء الإعداد، يطلب منك Appbox إنشاء أول حساب owner في n8n. أدخل عنوان البريد الإلكتروني للمسؤول وكلمة المرور والاسم الأول واسم العائلة التي تريد استخدامها لذلك الحساب.

عندما يصبح التطبيق جاهزاً، افتح Login URL الظاهر في صفحة التطبيق المثبت وسجّل الدخول بحساب owner الذي أنشأته للتو.

## إنشاء أول سير عمل

بعد دخولك إلى n8n، انتقل إلى **Overview** واختر **Create Workflow**. يفتح المحرر على لوحة تمثل فيها كل خطوة من الأتمتة node.

ابدأ بسير عمل يدوي بسيط:

1. انقر **Add first step**.
2. ابحث عن **Manual Trigger** وأضفه.
3. انقر زر الإضافة على node الخاص بالمشغّل.
4. أضف node ثانية، مثل **Edit Fields** أو **HTTP Request** أو node خاصة بتطبيق.
5. اضبط node في اللوحة الجانبية.

## اختبار سير العمل

انقر **Execute Workflow** لتشغيل سير العمل يدوياً.

بعد اكتمال التشغيل، انقر كل node لفحص البيانات التي استقبلتها وأعادتها. هذه أسرع طريقة لفهم ما يتحرك عبر سير العمل قبل ربط تطبيق حقيقي أو webhook أو مشغّل مجدول.

## ربط التطبيقات وبيانات الاعتماد

لتكاملات التطبيقات، أضف node التي تحتاج إليها وأنشئ credentials عندما يطلبها n8n.

للتطبيقات المستضافة على Appbox، النمط الشائع هو:

1. أنشئ API token أو webhook في التطبيق الذي تريد أتمتته.
2. أضف node المطابقة في n8n، أو استخدم node **HTTP Request** إذا لم توجد node مخصصة.
3. خزّن token في credentials داخل n8n.
4. اختبر بإجراء صغير واحد قبل إضافة branching أو retries أو خطوات إضافية.

للأحداث الواردة، استخدم trigger من نوع **Webhook**. يمنحك n8n عنوان webhook production يمكن للخدمات الخارجية استدعاؤه عبر HTTPS.

## أتمتة أولى بسيطة

جرّب هذا كأول سير عمل حقيقي:

1. أضف trigger من نوع **Webhook**.
2. أضف node **HTTP Request** لاستدعاء خدمة أخرى أو API لتطبيق Appbox.
3. أضف node **IF** للتفرع حسب الاستجابة.
4. أرسل رسالة إلى Mattermost أو Rocket.Chat أو قناة إشعارات أخرى.

يعطيك ذلك النمط الأساسي لكثير من أتمتات n8n: استقبل حدثاً، واجلب البيانات أو حوّلها، واتخذ قراراً، ثم أبلغ شخصاً أو حدّث نظاماً آخر.

## ملاحظة حول التوسع

صُممت حزمة Appbox كنشر n8n مدمج داخل حاوية واحدة مع تخزين دائم. إنها نقطة بداية ممتازة للأتمتة الشخصية وأتمتة الفرق والأتمتة الداخلية.

إذا كنت تخطط لتشغيل حجم تنفيذ عالٍ جداً، أو payloads كبيرة، أو مركز أتمتة بالغ الأهمية للأعمال، فاختبر الحمل بعناية وفكر في زيادة موارد التطبيق من صفحة التطبيق المثبت.

## ابدأ البناء

إذا كان لديك Appbox، فإن n8n من أسرع الطرق لجعل بقية أدواتك تبدو متصلة.

[شغّل n8n على Appbox](/appstore/app/264)، وأنشئ سير عمل، وشغّله مرة واحدة يدوياً، ثم حوّله إلى الأتمتة التي تتمنى أن تكون تطبيقاتك تملكها بالفعل.
