## Windows على Appbox

عندما أطلقنا VPSes في اليوم الثالث مع خمس توزيعات Linux، كان السؤال الطبيعي: "ماذا عن Windows؟" جعل Windows يعمل بشكل صحيح كـ VPS مُدار على Appbox كان تحدياً تقنياً كبيراً. تستخدم Linux VMs نظام virtiofs لمشاركة ملفات شبه أصلية بين المضيف والضيف. Windows لا يدعم virtiofs. لذلك بنينا تكاملاً مبنياً على Samba ينشئ مشاركات SMB مصادقاً عليها لكل VM، مما يمنح Windows الوصول السلس نفسه إلى بيانات Appbox الذي تحصل عليه Linux VPSes، لكن عبر بروتوكول يفهمه Windows أصلاً.

النتيجة هي آلة افتراضية كاملة بنظام Windows 11 Pro تعمل على KVM، مع تركيب بيانات Appbox الخاصة بك كقرص `Z:` (`APPBOX_DATA`). افتح File Explorer، وانتقل إلى `Z:\`، وسترى مجلدي `apps/` و`storage/` هناك.

## ما الذي يفتحه هذا

وجود Windows VPS على Appbox يفتح فئة جديدة تماماً من حالات الاستخدام:

- **برامج Windows فقط** - شغّل التطبيقات التي لا تملك إصدارات Linux ببساطة
- **تطوير .NET** - ابنِ واختبر تطبيقات .NET في بيئة Windows حقيقية
- **Office والإنتاجية** - شغّل Microsoft Office أو أدوات Adobe أو أي برنامج سطح مكتب
- **خوادم الألعاب** - استضف خوادم ألعاب تعمل على Windows فقط
- **سطح المكتب البعيد** - صِل إلى سطح مكتب Windows كامل من أي متصفح عبر وحدة تحكم الويب المدمجة، أو اتصل باستخدام Microsoft Remote Desktop لتجربة RDP أصلية
- **الاختبار وCI** - اختبر برنامجك على Windows من دون صيانة عتادك الخاص

هذا تثبيت Windows 11 Pro كامل مع وصول مسؤول. إذا كان يعمل على Windows، فهو يعمل هنا.

## توفر الخطط

لدى Windows 11 Pro متطلبات موارد عالية جداً ويستخدم I/O القرص بكثافة. لضمان تجربة جيدة للجميع، يتوفر Windows VPS حالياً على خطط Appbox المخصصة فقط (NG-18000 وما فوق). قد نوسع التوفر إلى خطط أخرى مستقبلاً مع استمرارنا في التحسين.

## الترخيص

لا يتضمن Windows 11 Pro VPS مفتاح منتج. سيعمل Windows غير مفعّل مع علامة مائية ثابتة على سطح المكتب، وإعدادات تخصيص محدودة، وتذكيرات دورية بالتفعيل، لكنه سيظل كامل الوظائف بخلاف ذلك. لإزالة هذه القيود، فعّله باستخدام مفتاح Windows 11 Pro صالح. نوصي بشراء واحد من [WinCDKey (~6 EUR)](https://wincdkey.com/product/microsoft-windows-11-professional-1-device-pc-software/).

## يعمل مع App Boost

يدعم Windows 11 Pro VPS **App Boost** من اليوم الثاني بالكامل. إذا احتجت إلى أداء أعلى للأحمال الثقيلة، حرّك مستوى boost للأعلى لتخصيص RAM وCPU أكثر لـ VM.

## App Freeze

هناك ميزة ثانية تُطلق اليوم وتنطبق على كل التطبيقات، وليس Windows فقط.

<p align="center">
  <img src="/blog-images/day5-windows-detail-freeze.png" alt="Windows 11 Pro VPS detail page showing App Boost and the new Freeze button" width="560" className="mt-8 mb-10" />
</p>

تتيح لك **App Freeze** إيقاف أي تطبيق مثبت مؤقتاً واستعادة app slots الخاصة به من دون فقدان أي بيانات. يتوقف التطبيق، وتبقى بياناته في مكانها تماماً، وتصبح slots التي كان يستخدمها متاحة لتطبيقات أخرى.

تحل هذه الميزة مشكلة تتكرر باستمرار: تريد تجربة تطبيق جديد، لكنك استخدمت كل slots. قبل اليوم، كان عليك إلغاء تثبيت شيء ما، وفقدان بياناته، فقط لتحرير slots كافية للتجربة. الآن يمكنك تجميد تطبيق، واستخدام تلك slots لتثبيت واختبار شيء جديد، ثم إلغاء تجميد التطبيق الأصلي عندما تنتهي. يعود كل شيء من حيث توقفت.

يوجد زر Freeze مباشرة في قسم Actions من صفحة تفاصيل أي تطبيق، إلى جانب Start وStop وRestart. يؤدي التجميد إلى إيقاف التطبيق، ووضع علامة frozen عليه، وإرجاع slots الخاصة به (بما في ذلك أي boost slots) إلى رصيدك المتاح. يعيد إلغاء التجميد التطبيق ويستهلك slots من جديد.

بعض الملاحظات المهمة:

- لا يمكنك تجميد تطبيق أثناء تثبيته أو تحديثه أو حذفه
- لا يمكنك إلغاء التجميد إذا لم يكن لدى Appbox الخاص بك slots شاغرة كافية لتغطية التكلفة
- تحتفظ التطبيقات المجمدة بكل بياناتها وإعداداتها سليمة
