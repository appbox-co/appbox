# Hostovaný aplikační server vs. VPS: co si vybrat?

Když lidé hledají vzdálený server, často používají několik různých výrazů pro stejný široký cíl: hostovaný aplikační server, cloudový hosting aplikací, soukromý hosting aplikací, vzdálený hosting aplikací, spravovaný hosting aplikací, VPS hosting.

Tyto výrazy se překrývají, ale nejsou totožné. Hostovaný aplikační server i VPS mohou dostat váš software online. Rozdíl je v tom, kolik systému chcete spravovat sami.

Tento průvodce rozebírá kompromis jednoduchými slovy, zvlášť pokud je vaším cílem provozovat aplikace jako [Plex](/apps/Plex), [Nextcloud](/apps/Nextcloud), [GitLab](/apps/GitLab), [WordPress](/apps/WordPress), File Browser nebo jiné self-hosted nástroje.

## Krátká verze

Zvolte **hostovaný aplikační server**, když chcete rychle instalovat a provozovat konkrétní aplikace s ovládáním z dashboardu, trvalým úložištěm, URL, přihlašovacími údaji a aktualizacemi řešenými přes tok platformy.

Zvolte **VPS**, když chcete plný root přístup k virtuálnímu stroji a vyhovuje vám samostatná správa operačního systému, balíčků, firewallu, služeb, mountů úložiště, záloh a řešení problémů.

Ani jedna možnost není univerzálně lepší. Každá je optimalizovaná pro jiný druh kontroly.

## Co je hostovaný aplikační server

Hostovaný aplikační server je serverové prostředí navržené kolem aplikací, ne kolem prázdného operačního systému.

Na Appboxu to znamená, že začínáte v [knihovně aplikací](/apps). Vyberete aplikaci, nainstalujete ji a spravujete ji z dashboardu. Appbox řeší okolní části platformy: URL aplikací, přihlašovací údaje, data aplikací, integraci úložiště, ovládání restartů, verze a přidělování zdrojů tam, kde je podporované.

## Co je VPS

VPS neboli virtual private server vám dá plný virtuální stroj. Obvykle si vyberete Linux distribuci, připojíte se přes SSH a spravujete stroj jako jakýkoli jiný server.

Ta dodatečná kontrola je skutečná. Můžete instalovat vlastní balíčky, provozovat Docker Compose, konfigurovat služby systemd, ladit firewall, spravovat uživatele a tvarovat server přesně podle sebe.

Appbox podporuje také plnohodnotné KVM VPSes pro případy, kdy takovou úroveň kontroly potřebujete. Spuštění a případy použití jsme popsali v článku [Launch Week Den 3 - Virtual Private Servers](/blog/cs/launch-week-day-3-vps).

## Kde vítězí hostovaný aplikační server

Hostovaný aplikační server se obvykle hodí víc, když vaše požadavky vypadají takto:

- Chcete rychle instalovat známé aplikace.
- Chcete URL aplikací, přihlašovací údaje, úložiště a restarty v jednom dashboardu.
- Raději se chcete vyhnout ruční údržbě reverse proxy.
- Chcete provozovat několik aplikací, které sdílejí stejné prostředí úložiště.
- Chcete ovládání na úrovni aplikací místo operací primárně přes SSH.
- Záleží vám víc na používání aplikace než na úpravách serveru pod ní.

Například pokud nastavujete [Plex](/apps/Plex), [Nextcloud](/apps/Nextcloud), [GitLab](/apps/GitLab) a [WordPress](/apps/WordPress), hostovaný aplikační server udrží pracovní postup zaměřený na aplikace. Každou nainstalujete ze stejného místa a spravujete je přes stejný ovládací panel.

To je časem důležité. První instalace je jen jedna část hostingu. Každodenní zkušenost tvoří restarty, aktualizace, kontroly zdrojů, přihlašovací údaje, domény, přístup k souborům, podpora a pamatování si, jak je všechno propojené.

## Kde vítězí VPS

VPS se obvykle hodí víc, když vaše požadavky vypadají takto:

- Potřebujete root přístup.
- Potřebujete vlastní systémové balíčky nebo chování na úrovni kernelu.
- Chcete provozovat vlastní Docker Compose stack.
- Potřebujete vlastní síťové rozložení nebo firewall.
- Chcete sami spravovat služby systemd.
- Stavíte něco, co nezapadá do modelu aplikace na jedno kliknutí.

Proto Appbox nabízí obě cesty. Hostované aplikace na jedno kliknutí jsou nejlepší, když je aplikační workflow známé. VPSes jsou nejlepší, když samotné prostředí musí zůstat otevřené.

Pokud stavíte vlastní aplikační stack, provozujete neobvyklé závislosti nebo experimentujete s infrastrukturou, VPS může být správný nástroj.

## Skrytá cena je provoz

Největší rozdíl není měsíční položka na faktuře. Je to provoz.

U VPS vlastníte celý řetězec:

- aktualizace operačního systému
- repozitáře balíčků
- pravidla firewallu
- konfiguraci Dockeru nebo služeb
- konfiguraci reverse proxy
- obnovování SSL certifikátů
- zálohy aplikací
- monitoring procesů
- čištění disku
- kontrolu logů
- bezpečnostní zpevnění

Někteří lidé to chtějí. Některé týmy to potřebují. Ale mnoho projektů hostingu aplikací ne.

U hostovaného aplikačního serveru platforma odstraňuje velkou část opakovaného nastavování. Pořád si musíte vybrat aplikace, rozumět úložišti a hlídat zdroje, ale nezačínáte pokaždé z prázdného shellu.

## Praktický rozhodovací seznam

Před výběrem si položte tyto otázky:

**Potřebujete root přístup?**
Pokud ano, zvolte VPS. Pokud ne, hostovaný aplikační server bude pravděpodobně jednodušší.

**Provozujete známou aplikaci z knihovny aplikací?**
Pokud ano, začněte hostovaným hostingem aplikací. VPS můžete kdykoli přidat později, pokud nastavení přeroste aplikační model.

**Chcete sami spravovat SSL, pravidla proxy a soubory služeb?**
Pokud ano, VPS. Pokud ne, hostované aplikace.

**Bude několik aplikací používat stejné soubory?**
Hostované aplikace na Appboxu jsou tady užitečné, protože data aplikací a sdílené úložiště jsou součástí modelu platformy.

**Experimentujete s vlastní infrastrukturou?**
Použijte VPS. Je to správný tvar pro otevřenou systémovou práci.

**Snažíte se rychle rozběhnout Plex, Nextcloud, GitLab nebo WordPress?**
Nejdřív použijte knihovnu aplikací.

## Jak Appbox kombinuje obojí

Praktické na Appboxu je, že z toho nemusíte dělat trvalé rozhodnutí identity. Můžete provozovat hostované aplikace pro věci, které zapadají do aplikačního modelu, a používat VPSes pro případy, kdy potřebujete plnou kontrolu nad strojem.

Obzvlášť užitečné je, že Appbox VPS má přístup k souborům z vašich hostovaných aplikací. VPSes mountují vaše Appbox data pod `/APPBOX_DATA/`, včetně:

- **`/APPBOX_DATA/apps/`** - data z vašich nainstalovaných hostovaných aplikací
- **`/APPBOX_DATA/storage/`** - vaše sdílené úložiště Appbox

To znamená, že vaše mediální aplikace, synchronizace souborů, dashboardy, trezor hesel a webové nástroje mohou žít ve spravovaném aplikačním toku, zatímco vlastní služby nebo vývojová prostředí mohou běžet na VPS, které stále vidí stejné soubory.

Například můžete provozovat Plex nebo Nextcloud jako hostované aplikace a pak použít Ubuntu VPS pro vlastní skripty, dávkové zpracování, zálohy, vývojové nástroje nebo cokoli dalšího, co těží z plné kontroly Linuxu. VPS nemusí být samostatný ostrov. Může pracovat se stejnými daty Appbox, která používají vaše hostované aplikace.

Pro mnoho lidí je takové hybridní nastavení klidnější než nutit všechno do jednoho modelu.

## Začněte pracovním postupem

Nejsnazší způsob, jak si vybrat, je začít pracovním postupem, ne typem serveru.

Pokud pracovní postup zní "chci provozovat tuto aplikaci", začněte v [knihovně aplikací Appbox](/apps).

Pokud pracovní postup zní "chci ovládat tento stroj", použijte VPS.

Obě možnosti jsou platné. Správná volba je ta, která vám umožní trávit víc času tím, co jste skutečně chtěli provozovat.
