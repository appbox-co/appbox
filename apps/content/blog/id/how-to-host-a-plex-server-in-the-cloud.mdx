# Cara Menghosting Server Plex di Cloud

Menjalankan Plex di rumah berjalan baik sampai tiba-tiba tidak lagi. Kecepatan upload, batas penyimpanan, kerusakan hardware, disk yang berisik, dan pengaturan akses remote semuanya bisa menghalangi tujuan sederhana: membuka library media Anda dan menekan play.

Server Plex cloud menyelesaikan masalah yang berbeda: Anda mendapatkan server media pribadi yang selalu online, bisa dijangkau dari mana saja, dan tidak terikat ke mesin di bawah meja Anda. Trade-off-nya adalah Anda ingin setup yang sederhana, mudah diprediksi, dan dibangun di sekitar aplikasi media, bukan server kosong generik.

Panduan ini membahas hal-hal yang penting saat memilih setup Plex di cloud dan bagaimana Appbox masuk ke workflow tersebut.

## Yang Dibutuhkan Server Plex Cloud

Plex tidak terlalu rumit, tetapi server Plex hosted yang baik tetap membutuhkan beberapa hal agar nyaman digunakan sehari-hari:

- **Penyimpanan andal** untuk library media, metadata, poster, dan data aplikasi Anda.
- **CPU dan memori yang cukup** untuk scanning library, akses remote, dan transcoding sesekali.
- **Akses jaringan cepat** agar pemutaran remote mulai dengan cepat dan tetap stabil.
- **URL persisten** untuk aplikasi web Plex dan akses server.
- **Manajemen aplikasi yang sederhana** agar update, restart, dan kredensial tidak menjadi proyek akhir pekan.
- **Ruang untuk menjalankan aplikasi pendamping** seperti Jellyfin, Emby, Tautulli, Overseerr, [File Browser](/apps/File%20Browser%20%26%20Webdav), atau Nextcloud.

Anda bisa membangun semuanya sendiri di VPS. Banyak orang melakukannya. Tetapi itu berarti Anda juga bertanggung jawab atas sistem operasi, stack Docker, firewall, reverse proxy, sertifikat SSL, mount penyimpanan, restart layanan, dan setiap detail operasional kecil yang menyertainya.

Appbox dirancang untuk jalur lain: pilih aplikasi, instal, lalu kelola dari dashboard.

## Mengapa Menggunakan Appbox untuk Hosting Plex

Di Appbox, Plex tersedia sebagai aplikasi sekali klik. Alih-alih memulai dari server kosong, Anda memulai dari environment aplikasi terkelola yang sudah memahami domain, data aplikasi, penyimpanan, alokasi resource, dan kontrol dashboard.

Itu memberi Anda beberapa manfaat praktis:

- **Instal sekali klik** dari library aplikasi Appbox.
- **Kontrol dashboard** untuk detail aplikasi, restart, kredensial, versi, dan penggunaan resource.
- **Penyimpanan bersama** di seluruh aplikasi Appbox Anda, sehingga tool terkait dapat bekerja dengan file yang sama.
- **Custom domain dan HTTPS** ditangani melalui flow platform.
- **App Boost** saat aplikasi yang lebih berat membutuhkan lebih banyak CPU atau memori dari paket Anda.
- **Aplikasi pendamping** tersedia dari library yang sama, termasuk [Jellyfin](/apps/Jellyfin), [Emby](/apps/Emby), [Tautulli](/apps/Tautulli), [File Browser](/apps/File%20Browser%20%26%20Webdav), dan [Nextcloud](/apps/Nextcloud).

Jika tujuannya adalah menjalankan Plex, bukan menjadi tim operasi untuk stack media Anda sendiri, detail-detail itu penting.

## Hosting Plex vs VPS Generik

VPS itu fleksibel. Anda mendapatkan akses root dan bisa menginstal hampir apa saja. Itu berguna saat Anda membutuhkan kontrol penuh, layanan custom, atau environment Linux yang sangat spesifik.

Untuk hosting Plex, fleksibilitas juga bisa berubah menjadi hambatan. Setup VPS biasa berarti memilih distribusi Linux, menginstal Docker atau package, mengonfigurasi penyimpanan, menyiapkan reverse proxy, mengelola SSL, membuka port, membuat backup, dan memantau penggunaan resource sendiri.

Appbox lebih opinionated. Tujuannya bukan menggantikan setiap use case VPS. Tujuannya adalah membuat aplikasi hosted lebih mudah.

Pilih VPS jika Anda ingin mengelola setiap lapisan sendiri.

Pilih Appbox jika Anda menginginkan environment aplikasi media pribadi dengan Plex dan aplikasi terkait sebagai workflow utama.

## Memilih Paket Appbox yang Tepat

Paket yang tepat bergantung pada apa lagi yang ingin Anda jalankan di samping Plex.

Untuk setup ringan, Anda mungkin hanya membutuhkan Plex plus file manager. Untuk environment media yang lebih lengkap, Anda mungkin menginginkan Plex, Tautulli, File Browser, Nextcloud, dan beberapa tool pendukung. Jika Anda memperkirakan scanning yang lebih berat, library yang lebih besar, atau beberapa aplikasi aktif sekaligus, sisakan kapasitas paket yang cukup sejak awal.

Hal yang berguna tentang Appbox adalah paketnya bukan sekadar satu slot aplikasi. Paket adalah sebuah environment. Anda bisa menginstal beberapa aplikasi, mengalokasikan resource dengan App Boost jika didukung, dan menjaga layanan terkait tetap dekat dengan penyimpanan yang sama.

Ini sangat membantu saat server media Anda berkembang dari "hanya Plex" menjadi stack aplikasi pribadi kecil.

## Menyiapkan Plex di Appbox

Flow dasarnya sederhana:

1. Buka [halaman aplikasi Plex](/apps/Plex).
2. Pilih paket Appbox dengan kapasitas yang cukup untuk library dan aplikasi pendamping Anda.
3. Instal Plex dari flow aplikasi.
4. Buka halaman detail aplikasi untuk menemukan URL dan kredensial yang dibuat.
5. Masuk ke Plex dan hubungkan ke akun Anda.
6. Tambahkan folder media Anda dan biarkan Plex memindai library.

Dari sana, Anda bisa menambahkan aplikasi terkait dari library aplikasi Appbox sesuai kebutuhan.

## Aplikasi Pendamping yang Layak Dipertimbangkan

Plex sering menjadi pusat setup, tetapi jarang menjadi satu-satunya aplikasi yang berguna.

[Tautulli](/apps/Tautulli) berguna untuk monitoring Plex dan aktivitas library. [Jellyfin](/apps/Jellyfin) dan [Emby](/apps/Emby) adalah server media alternatif jika Anda ingin membandingkan workflow atau menjalankan opsi open-source berdampingan dengan Plex. [File Browser](/apps/File%20Browser%20%26%20Webdav) memberi Anda file manager web yang sederhana. [Nextcloud](/apps/Nextcloud) berguna jika Anda menginginkan penyimpanan cloud pribadi dan sinkronisasi file di environment Appbox yang sama.

Di sinilah Appbox menjadi lebih dari sekadar "host Plex." Ini adalah environment aplikasi hosted untuk tool lain di sekitar library media Anda.

## Beberapa Tips Praktis

Jaga struktur library tetap rapi sejak awal. Plex cukup baik dalam mencocokkan media, tetapi folder yang bersih dan penamaan yang konsisten menghemat waktu nantinya.

Gunakan direct play jika memungkinkan. Transcoding bisa berguna, tetapi lebih berat daripada streaming file dalam format yang sudah didukung perangkat Anda.

Pantau pertumbuhan penyimpanan. Poster, metadata, thumbnail, dan aplikasi pendamping semuanya mengonsumsi ruang dari waktu ke waktu, bukan hanya file medianya sendiri.

Jaga aplikasi pendamping tetap fokus. Menginstal semuanya sekaligus memang menggoda, tetapi setup yang lebih kecil lebih mudah dipahami dan dituning.

## Mulai dari Halaman Aplikasi Plex

Jika Anda menginginkan server Plex pribadi tanpa membangun seluruh stack secara manual, mulai dari [halaman aplikasi Plex](/apps/Plex). Anda bisa membandingkan paket, memeriksa kebutuhan aplikasi, dan menginstalnya melalui flow Appbox yang sama seperti library aplikasi lainnya.

Dan jika nanti Anda ingin berkembang melampaui Plex, environment yang sama bisa menghosting aplikasi lain yang membuat server media pribadi lebih berguna.
