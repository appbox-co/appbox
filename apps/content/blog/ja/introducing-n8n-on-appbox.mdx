# Appbox で n8n が利用可能になりました

n8n が Appbox で利用できるようになりました。

つまり、プライベートなワークフロー自動化ワークスペースをデプロイし、ビジュアルエディターを開いて、アプリ、API、スケジュール、webhook、カスタムコードをつなぎ始められます。サーバーを手作業で管理する必要はありません。

[Appbox App Store から n8n をインストール](/appstore/app/264)。

## 作れるもの

あるアプリが別のアプリに反応したり、データを整えたり、API を呼び出したり、通知を送ったり、スケジュールされたタスクを実行したりする場面で、n8n は役立ちます。

Appbox では、webhook、API、通知フック、WebDAV、RPC、スクリプト可能なインターフェースを持つアプリの自然な相棒になります。

AI、サポート、ビジネスワークフロー向け：

- **OpenClaw** は自動化の AI レイヤーとして動かせます。サポートチケット、監視アラート、ファイルアップロードの要約が必要になったら n8n から webhook を送り、OpenClaw に返信の下書き、依頼の分類、イベントのタスク化を任せられます。
- **Chatwoot** はカスタマーサポート向けの受信箱イベントと API を公開しています。緊急の会話にタグを付けたり、別システムから新規連絡先を補完したり、VIP から連絡があったときにチームチャンネルへ通知したり、会話終了後にフォローアップタスクを作成したりできます。
- **Mautic** には REST API とキャンペーン自動化の仕組みがあります。フォームからリードを同期し、プロダクトイベント後に連絡先をスコアリングし、webhook 送信からキャンペーンへ追加し、リードがしきい値を超えたら営業に通知できます。
- **ERPNext** は API を通じて業務レコードを公開します。フォーム送信から請求書を作成し、支払いが入ったら商談を更新し、在庫が少なくなったら運用チームへ通知し、顧客レコードを別の Appbox アプリへ送れます。
- **Plane** と **OpenProject** はタスクの送信先としてよく合います。インシデント、フォーム送信、失敗したジョブ、顧客リクエストを、適切な担当者と文脈付きの課題や作業パッケージに変換できます。
- **Rallly** は日程調整ワークフローに組み込めます。リクエストフォームから投票を作成し、会議時刻が決まったら参加者へ通知し、回答期限前にリマインダーを送れます。
- **OnlyOffice** と **Jitsi** は文書と会議のワークフローに向いています。文書の準備ができたらレビュー担当者に通知し、承認後に会議リンクを作成し、会議後のフォローアップタスクを Plane や OpenProject に送れます。

コンテンツ、公開、開発者ワークフロー向け：

- **EmDash** はフォーム、短縮リンク、プラグイン、自動ビルドフックを備えた構造化 CMS です。フォーム送信を n8n に送り、コンテンツ公開時に編集者へ通知し、下書き公開後に再ビルドを起動し、新しい投稿を Mattermost、Rocket.Chat、ソーシャル投稿キューへ送れます。
- **WordPress** には REST API とフックのエコシステムがあります。フォーム送信から投稿の下書きを作成し、コンテンツ公開時に告知を出し、メディアを別のストレージアプリへバックアップし、予約投稿の前に編集リマインダーを送れます。
- **GitLab**、**Gogs**、その他の Git アプリは、リポジトリ、プロジェクト、システムのフックを公開します。タグに応じたリリース通知、マージ後のデプロイチェックリスト作成、失敗したパイプライン概要のチャット投稿、バグ報告テンプレートからのプロジェクト管理課題作成ができます。
- **Hugo Git** は Git ベースの公開フローに置けます。コンテンツコミット後に再ビルド通知を起動し、新しいタグが付いたら変更履歴を公開し、各プルリクエストのレビュータスクを n8n に作らせられます。
- **Docmost** はドキュメントの送信先として便利です。新規顧客登録時にオンボーディングチェックリストを作成し、監視イベントからインシデントメモを投稿し、ワークフロー出力から社内ランブックを更新し続けられます。
- **Jupyter Notebook** はデータとレポートのワークフローを支援できます。スケジュールされた notebook を実行し、データベース更新後にレポートをエクスポートし、長時間の分析が終わったらチャンネルへ通知できます。
- **Serposcope** は SEO ワークフローにデータを供給できます。順位変動アラートを送り、週間の検索順位ダイジェストを作成し、重要なキーワードが下がったらコンテンツタスクを開けます。
- **MySQL/MariaDB**、**Nginx & PHP**、**LAMP Stack** は社内ツールに役立ちます。n8n で行を書き込み、非公開の管理エンドポイントを呼び出し、定期メンテナンスチェックを実行し、カスタムアプリに新しい送信があったらチームへ通知できます。

ファイル、ストレージ、同期、バックアップ向け：

- **Nextcloud** は WebDAV と広いアプリエコシステムを公開します。アップロードされたファイルをレビュー用フォルダへ移動し、クライアントのアップロード到着時にチームへ通知し、カレンダーのフォローアップを作成し、承認済み文書をプロジェクトスペースへコピーできます。
- **SFTPGo** には REST API、イベントアクション、ログインフックがあります。クライアントがレポートをアップロードしたらファイル処理ワークフローを起動し、完了したエクスポートを Nextcloud へ移動し、パートナーの転送到着時にチャンネルへ通知し、期限後に一時ユーザーを無効化できます。
- **File Browser Quantum**、**File Browser WebDAV**、**Kodbox** は、REST API、WebDAV、コマンドフック、ストレージプラグインなどのファイル管理面を公開します。共有フォルダを監視し、メタデータを生成し、完成した素材をアーカイブストレージへコピーし、公開前にファイルを承認に回せます。
- **Rclone** はオブジェクトストレージ、SFTP、WebDAV、クラウドリモートをまたいでスクリプト化できます。n8n から同期ジョブをスケジュールし、大規模移行後に転送を検証し、処理済みメディアを長期保存へ移動し、失敗概要をチャットへ送れます。
- **Restic REST Server** は restic クライアント向けの HTTPS バックアップ先を提供します。定期バックアップチェックの失敗時に管理者へ通知し、スナップショット欠落後にインシデントを開き、日次のバックアップ健全性概要を Docmost に書き込めます。
- **Syncthing** と **Resilio Sync** はファイルイベントのパイプラインに便利です。フォルダ同期の完了に反応し、完了したドロップをメディアライブラリへコピーし、期待されたファイルが届いていないときにユーザーへ警告できます。
- **Pure-FTPd** とその他の FTP/SFTP 形式のエンドポイントは、レガシーなファイルワークフローに組み込めます。新しいパートナーファイルをポーリングし、SFTPGo や Nextcloud へ移動して、処理完了後に適切なチームへ通知できます。

セキュリティ、アクセス、インフラ向け：

- **Vaultwarden** は Bitwarden 互換 API と任意の SMTP 通知を公開します。保管庫レビューのリマインダーを作成し、オンボーディングやオフボーディングのタスク期限を管理者へ通知し、チーム横断のパスワードローテーションチェックリストを調整できます。
- **OpenVPN** と **WireGuard VPN** はアクセスワークフローの一部にできます。認証情報のローテーションが必要なときにオペレーターへ通知し、新しい VPN ユーザー向けのチェックリストタスクを作成し、n8n から定期接続チェックを実行できます。
- **Ubuntu**、**Debian**、**Fedora**、**Rocky Linux**、**AlmaLinux**、**Arch Linux**、**Windows VPS** アプリはスクリプト可能な環境を提供します。n8n で SSH コマンドを起動し、健全性チェックを実行し、レポートファイルを収集し、スケジュール済みジョブ後にメンテナンス概要を投稿できます。
- **Ubuntu**、**Debian**、**Firefox remote desktop** アプリはブラウザやデスクトップ作業を支援できます。手動レビュー作業のリマインダーを予定し、スクリプト出力を取得し、完了したデスクトップエクスポートをファイルストレージへ引き渡せます。

監視、チャット、通知向け：

- **Uptime Kuma** は豊富な通知先と webhook をサポートします。ダウンタイムイベントを Mattermost や Rocket.Chat のアラートに変換し、OpenProject のインシデントを作成し、OpenClaw に短いインシデント概要を依頼し、すべての復旧をステータス用スプレッドシートに記録できます。
- **Mattermost** と **Rocket.Chat** は通知と承認のエンドポイントとして便利です。任意のワークフローから読みやすい更新を送り、チームにリクエスト承認を求め、誰かが応答したら自動化を続行できます。
- **RocketChat Hubot** はチャットコマンドに便利です。チームメイトがチャットから n8n webhook を起動したり、デプロイ状況を尋ねたり、監査証跡付きのスクリプト化メンテナンスワークフローを開始したりできます。
- **Matterbridge** は IRC、Matrix、XMPP、Slack、Discord、Microsoft Teams、Telegram などを接続します。n8n で運用イベントを一度整形し、それを見る必要があるコミュニティやルームへ中継できます。
- **ZNC** と **The Lounge** は chat-ops ワークフローの一部にできます。重要な IRC イベントを現代的なチームチャンネルへ転送し、ハイライトをアーカイブし、監視中のキーワードが現れたらオペレーターへ通知できます。

メディアとダウンロード自動化向け：

- **Tautulli** は Plex のアクティビティ、通知スクリプト、JSON API を公開します。新規追加メディアのダイジェストを送り、ストリームエラー時に警告し、利用レポートをエクスポートし、ストレージが上限を超えたらクリーンアップタスクを作成できます。
- **Plex**、**Jellyfin**、**Emby**、**Navidrome**、**Audiobookshelf**、**Calibre**、**Madsonic**、**SeAT** は、ライブラリやコミュニティのワークフローにデータを供給できます。新しいメディアを告知し、インポート後にライブラリを更新し、視聴や読書のダイジェストを作成し、メタデータをダッシュボードへ同期し、追跡中のアクティビティが変化したらコミュニティへ通知できます。
- **Immich** は写真ワークフローに合います。共有アルバムが変わったら家族へ通知し、エクスポートしたアルバムをアーカイブストレージへコピーし、大量のモバイルアップロード後にバックアップチェックを起動できます。
- **AzuraCast** には REST API と webhook があります。曲やライブ番組の変更を告知し、Web サイトのウィジェットを更新し、予定放送の前にホストへ通知し、番組後にプレイリストのメタデータをアーカイブできます。
- **Icecast** は放送ステータスのワークフローを支援できます。ストリームの可用性を監視し、現在再生中の更新を投稿し、ソースが切断されたらチャンネルへ警告できます。
- **Flood**、**ruTorrent**、**qBittorrent**、**Transmission**、**Deluge** は RPC または Web API を公開します。完了したダウンロードを監視し、完成ファイルを適切なライブラリへ移動し、メディアスキャンを起動し、リクエストされた項目の準備ができたらユーザーへ通知できます。
- **PufferPanel**、**TeamSpeak 3**、ゲーム/チャットサーバーも運用ワークフローに組み込めます。サーバーに対応が必要なときに管理者へ通知し、メンテナンスリマインダーを作成し、予定された変更後にプレイヤー向け告知を投稿できます。

組み込みノード、webhook、HTTP リクエスト、データベース接続、コードノードを通じて、n8n を外部サービスと組み合わせることもできます。

## n8n をインストールする

[n8n App Store ページ](/appstore/app/264)を開き、Appbox を選んでインストールを開始します。

セットアップ中に、Appbox は n8n の最初のオーナーアカウントを作成するよう求めます。そのアカウントで使う管理者メールアドレス、パスワード、名、姓を入力してください。

アプリの準備ができたら、インストール済みアプリページに表示される Login URL を開き、先ほど作成したオーナーアカウントでサインインします。

## 最初のワークフローを作成する

n8n に入ったら、**Overview** に移動して **Create Workflow** を選びます。エディターがキャンバス上で開き、自動化の各ステップがノードとして表示されます。

まずはシンプルな手動ワークフローから始めましょう：

1. **Add first step** をクリックします。
2. **Manual Trigger** を検索して追加します。
3. トリガーノードのプラスボタンをクリックします。
4. **Edit Fields**、**HTTP Request**、またはアプリ固有のノードなど、2 つ目のノードを追加します。
5. サイドパネルでノードを設定します。

n8n のメインワークスペースはキャンバスです。ここでノードを追加し、接続し、ワークフローを整え、テスト実行を行います。

## ワークフローをテストする

**Execute Workflow** をクリックして、ワークフローを手動で実行します。

実行が完了したら、各ノードをクリックして受け取ったデータと返したデータを確認します。実際のアプリ、webhook、スケジュールトリガーにつなぐ前に、ワークフロー内で何が流れているかを理解する最短の方法です。

## アプリと認証情報を接続する

アプリ連携では、必要なノードを追加し、n8n に求められたら認証情報を作成します。

Appbox でホストされているアプリでは、よくある流れは次のとおりです：

1. 自動化したいアプリで API トークンまたは webhook を作成します。
2. 対応する n8n ノードを追加します。専用ノードがない場合は **HTTP Request** ノードを使います。
3. トークンを n8n の認証情報に保存します。
4. 分岐、リトライ、追加ステップを入れる前に、小さなアクション 1 つでテストします。

受信イベントには **Webhook** トリガーを使います。n8n は外部サービスが HTTPS 経由で呼び出せる本番用 webhook URL を提供します。

## 最初のシンプルな自動化

最初の実用ワークフローとして、これを試してみてください：

1. **Webhook** トリガーを追加します。
2. 別のサービスまたは Appbox アプリ API を呼び出す **HTTP Request** ノードを追加します。
3. レスポンスに応じて分岐する **IF** ノードを追加します。
4. Mattermost、Rocket.Chat、または別の通知チャンネルへメッセージを送ります。

これにより、多くの n8n 自動化に共通する基本パターンができます。イベントを受け取り、データを取得または変換し、判断してから、人に通知するか別のシステムを更新します。

## スケールについて

Appbox パッケージは、永続ストレージ付きのコンパクトな単一コンテナ n8n デプロイとして設計されています。個人、チーム、社内の自動化を始めるには最適です。

非常に多い実行量、大きなペイロード、またはビジネス上重要な自動化ハブを運用する予定がある場合は、ワークロードを慎重にテストし、インストール済みアプリページからアプリリソースの増強を検討してください。

## 作り始めましょう

Appbox をお持ちなら、n8n は他のツール同士をすばやくつながっているように感じさせる方法のひとつです。

[Appbox に n8n をデプロイ](/appstore/app/264)し、ワークフローを作成して一度手動で実行し、それをあなたのアプリに最初から欲しかった自動化へ育ててください。
