# Launch Week Day 1: 新しいダッシュボードが公開されました

Launch Week は月曜日に始まるとお伝えしました。そして Day 1 が公開されました。

本日、新しい Appbox ダッシュボード体験をリリースします。これはコントロールパネル全体にわたる大きな UI とワークフローのアップグレードで、目的はシンプルです。日々の appboxes 運用を、より速く、より分かりやすく、より煩わしくなくすることです。

このアップデートは長い間進めてきたもので、ようやく皆さんに届けられることをとても嬉しく思います。

## 新しいコントロールセンター

これまで、この情報は複数のページやフローに分散していました。今日からは、それらを1つの運用ビューにまとめる専用ダッシュボードを導入します。

- 最新のブログとプラットフォーム更新
- サービス概要カード

- 最新および最近更新されたアプリ

タブを行き来する代わりに、1ページを開くだけで重要な情報をすぐに確認できます。

## カスタムドメインが主要機能に

カスタムドメインは、もう気まずい追加手順ではありません。インストール体験の中核になりました。

アプリがドメインを必要とする場合、セットアップはインストールダイアログ内で、分かりやすいガイド付きフローとして行われます。

1. Appbox を選択します。
2. **Domain Type** を選びます。
   - **Appbox Domain** は Appbox の既定ドメインを使います。
   - **Custom Domain** は所有している独自ドメインを使います。
3. **Base Domain** を選びます。
4. 使用したい **Subdomain** を入力します。
5. カスタムドメインを選んだ場合、Appbox は追加すべき正確な DNS レコード（host/name と target IP）を表示し、その場で検証できます。

レコードを手探りで推測したり、ドキュメントをコピーしたり、インストールを何度もやり直したりする代わりに、アプリをインストールする同じ場所で、組み込みの DNS 設定と検証ステップを使えます。

## 新しい File Explorer（ゼロから構築）

File Explorer はゼロから作り直しました。以前の体験では、ファイル管理は削除に限られていました。今では、次の機能を備えた本格的なファイルマネージャーです。

- アップロード
- 新規フォルダ作成
- 名前変更
- 削除
- ダウンロード
- リスト/グリッド表示の切り替え
- パンくずナビゲーションとコンテキストアクション

追加のアプリをインストールしなくても、appbox ファイルをネイティブに管理できます。

## 二要素認証

2026年において、セキュリティは任意であるべきではありません。本日から、Appbox アカウントは組み込みの二要素認証に対応します。

アカウントエリアから直接 2FA を有効化・管理でき、アカウント復旧用のリカバリーコード管理も含まれます。

## ダッシュボード全体の新しいデザイン言語

ナビゲーション、余白、カード、タイポグラフィ、インタラクション状態にわたって、ビジュアルシステムを刷新しました。目標は一貫性と集中です。視覚的なノイズを減らし、階層を強め、読みやすさを高め、ダークテーマとライトテーマの両方でより現代的なプロダクト感を実現しました。

真夜中にアプリをインストールするとき、まぶしい白画面に悩まされることはもうありません。

- UI 内で直接使える **ネイティブな言語切り替え**（今後さらに多くの言語を追加予定）。
- サービスカード上に小さなクイックアクセスアイコンとして表示される **ピン留めアプリのショートカット**。よく使うアプリへより速く移動できます。
