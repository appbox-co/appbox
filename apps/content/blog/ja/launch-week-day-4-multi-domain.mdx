# Launch Week Day 4: VPS アプリの複数ドメイン

Day 4。昨日はフル仮想マシンを提供しました。今日は、それにふさわしいドメインサポートを提供します。

Day 3 に VPS をリリースしたとき、各 VM には1つのドメインが付きました。多くの用途ではそれで十分ですが、VPS 内で Web サーバーを動かしている場合、そして多くの方がそうしていますが、1つのドメインでは足りません。同じマシンに複数のドメインを向け、それぞれに SSL 証明書と Web サーバー設定が必要です。

本日、VPS アプリは **複数ドメイン** に対応しました。必要なだけ `*.user.appboxes.co` サブドメインを追加し、独自のカスタムドメインを持ち込み、または両方を混在させられます。各ドメインには SSL と、すぐ使える Web サーバー設定が付きます。

## 仕組み

実行中の任意の VPS アプリを開くと、詳細ページに **Custom Domains** カードがあります。ここがすべてのドメインの場所です。その VPS に紐づく全ドメイン、どれがプライマリか、いつ追加されたかを表示する、検索・並び替え可能なテーブルです。

<p align="center">
  <img src="/blog-images/day4-domains-table.png" alt="VPS アプリ詳細ページの Custom Domains テーブル" width="860" className="mt-8 mb-10" />
</p>

新しいドメインを追加するには、**Add Domain** をクリックします。親ドメインを選び、サブドメインを入力すれば、あとはシステムが処理します。

## Appbox サブドメイン

ドメインを追加する最速の方法は、既存の Appbox ドメインを使うことです。`user.appboxes.co` ドメインを親として選び、サブドメインを入力すれば完了です。DNS はすでに処理されており、ドメインは即座に検証されます。

これは、素早いプロジェクト URL、ステージング環境、または VPS 上の異なるサービスにそれぞれ専用アドレスを与えるのに最適です。backend 用に `api.user.appboxes.co`、frontend 用に `app.user.appboxes.co` が必要ですか？1分以内に両方追加できます。

<p align="center">
  <img src="/blog-images/day4-add-subdomain.png" alt="VPS に Appbox サブドメインを追加する画面" width="560" className="mt-8 mb-10" />
</p>

これらは既存の Appbox ワイルドカード証明書の配下にあるため、SSL は自動です。待ち時間も、証明書生成も、設定も不要です。

## カスタムドメイン

所有している独自ドメインを VPS に向けることもできます。カスタムベースドメインを選択し（またはダイアログから直接新規追加し）、サブドメインを入力すると、UI が DNS 設定を案内します。

ダイアログには、作成すべき正確な A レコードが表示されます。ホスト名、IP アドレス、すべてです。Cloudflare やその他の人気 DNS プロバイダー向けのプロバイダー別手順もあるため、A レコードの追加方法を調べる必要はありません。

<p align="center">
  <img src="/blog-images/day4-add-custom-domain.png" alt="DNS 検証とプロバイダー別手順付きでカスタムドメインを追加する画面" width="560" className="mt-8 mb-10" />
</p>

DNS が正しい IP を指したら、**Verify Again** を押します。システムはドメインを追加する前に、レコードが正しいことを確認します。

## SSL と Web サーバー設定

追加したすべてのドメインに SSL が付きます。Appbox サブドメインでは既存のワイルドカード証明書を使用します。カスタムドメインでは、証明書が自動的に発行されます。

VPS ユーザーにとって本当に便利なのはここです。各ドメインには **View Config** ボタンがあり、Nginx と Apache の両方に対応した、貼り付けるだけの設定スニペットが表示されます。設定には正しい `server_name`、SSL 証明書パス、cipher suites、protocol settings が含まれます。スニペットをコピーし、VM 内の Web サーバー設定に貼り付け、reload すれば、そのドメインでトラフィックを配信できます。

<p align="center">
  <img src="/blog-images/day4-ssl-config.png" alt="Nginx と Apache 向け SSL 設定スニペット" width="560" className="mt-8 mb-10" />
</p>

ファイルパスを推測する必要はありません。証明書の場所を探し回る必要もありません。cipher 文字列を調べる必要もありません。すべて用意されていて、コピーするだけです。

## VPS ユーザーにとってなぜ重要か

マルチドメイン対応のない VPS は、1つの住所しか持たないサーバーです。これはすぐに明らかになる形で制限になります。

- 別 API を持つ Web アプリを動かしていますか？少なくとも2つのドメインが必要です。
- VM 内で複数サイトをホストしていますか？それぞれに独自ドメインが必要です。
- production と並べて staging 環境が欲しいですか？それも別ドメインです。
- ローカル開発中に本物のドメインでテストしたいですか？さらに1つ必要です。

## Day 4 は今すぐ利用できます

複数ドメイン対応は、すべての VPS アプリで今すぐ利用できます。VPS を開き、Custom Domains までスクロールして、追加を始めてください。
