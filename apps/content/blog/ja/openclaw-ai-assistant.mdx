# OpenClaw が Appbox に登場

以前から個人的にとても楽しみにしていたものを、App Store に追加しました。

[OpenClaw](https://openclaw.ai) は、自分のインフラ上で完全に動作するオープンソース AI アシスタントです。クラウド AI アシスタントのプライベートな self-hosted 代替と考えてください。ただし、これはあなたの Appbox 上に存在し、会話を記憶し、実際にあなたのために作業できます。

普段使っているチャットアプリから話しかけられます。WhatsApp、Telegram、Discord、Slack、Signal、さらには iMessage まで。Anthropic、OpenAI、Google、xAI、Mistral などのモデルの力で応答します。自分の API キーを持ち込み、モデルを選び、すべては自分のサーバー上に残ります。

それだけでも便利です。ですが、静かにストアへ追加するだけでなくブログ記事を書いている理由は、これを Appbox の他の部分と組み合わせたときに起きることにあります。

## 他のアプリを見ることができます

Appbox 上の OpenClaw が他の場所で動かす場合と違う理由はここにあります。

すべての Appbox には共有データレイヤーがあります。インストール済みアプリは共通マウント配下にそれぞれディレクトリを持ち、任意のアプリが読み書きできる共有ストレージ領域もあります。OpenClaw はこの同じ共有ファイルシステムへアクセスできます。つまり、壊れやすい API 連携ではなく、直接のファイルシステムアクセスによって、他のインストール済みアプリのデータと一般ストレージを見ることができます。

実際には、あなたの AI アシスタントは、文脈のない自分専用コンテナに隔離されているわけではありません。ファイル、データベース、メディア、コード、設定のすぐ隣にいます。

## 他のアプリにログインできます

共有ファイルシステムに加え、OpenClaw にはフル headless browser（Puppeteer 付き Chromium）と、一般的なデータベースクライアントおよび CLI ツールが組み込まれています。他のアプリのログイン情報を渡せば、実際にサインインしてあなたの代わりに操作できます。

Docmost のチームドキュメントを更新したいですか？OpenClaw に書く内容を伝え、URL と認証情報を渡してください。ログインし、適切なページへ移動し、編集を行います。Nginx & PHP スタックに新しいランディングページが必要ですか？欲しい内容を説明すれば、OpenClaw が HTML、CSS、JavaScript を書き、共有ファイルシステム経由で Web root へ直接デプロイできます。WordPress を実行していますか？ブログ記事の下書きと公開、プラグイン設定の更新、ページのスタイル変更を頼めます。あなたと同じように管理ダッシュボードを操作できます。

これは Web UI に限られません。OpenClaw には `mysql`、`psql`、`redis-cli`、`mongosh`、SSH サポートが含まれているため、渡した認証情報を使ってデータベースやサービスへ直接接続できます。重要なのは、OpenClaw がアプリごとの特別な連携やプラグインを必要としないことです。あなたがログインできるものには、アシスタントもログインできます。

## 実際にはどんなことができるか

AI アシスタントが Appbox エコシステムへアクセスできると何ができるのか、もう少し具体例を見ていきます。

### メディアサーバーの整理

Plex、Jellyfin、Emby を実行している場合、OpenClaw には `ffmpeg`、`mediainfo`、`exiftool` が組み込まれています。これらは、ディレクトリを手作業で掘らなくても、ファイルメタデータを確認したりエンコード問題を見つけたりするのに便利です。フォルダーの中身を一覧させたり、形式を確認させたり、メディアサーバーが正しくインデックスできるよう命名規則の整理を手伝わせたりできます。

### 苦痛のないデータベース作業

OpenClaw のコンテナには、`mysql`、`psql`、`sqlite3`、`redis-cli`、`mongosh` がプリインストールされています。MariaDB を実行している場合、OpenClaw にデータベースを問い合わせ、schema を調査し、遅い query をデバッグし、migration の作成を手伝わせることができます。すべてスマートフォンのチャットメッセージから行えます。

WordPress や MariaDB をバックエンドに持つ別の CMS を実行していますか？OpenClaw にデータベースの健全性確認、孤立した post metadata の検索、特定コンテンツの export を依頼できます。SSH で入ったり、正しい CLI flags を覚えたりする必要はありません。必要なことを普通の言葉で説明するだけです。

### GitLab とコードワークフロー

GitLab をインストールしている場合、OpenClaw はリポジトリを直接扱えます。`git` が組み込まれており、高速なコード検索のために `ripgrep` と `fd` もあります。リポジトリ全体から特定パターンを検索したり、最近のコミットを要約したり、見慣れないコードの理解を手伝わせたりできます。

内部プロジェクトに GitLab を使っているチームにとって、コードベースを実際に読んで質問に答えられる AI アシスタントを、コードをサードパーティのクラウドサービスへ送らずに使えることは、開発ワークフローの大きな改善です。

### ファイル整理と同期

Nextcloud、SFTPGo、File Browser を実行していますか？OpenClaw は、それらのサービスを流れるファイル管理を手伝えます。ドキュメントを検索し、ディレクトリ構造を整理し、重複を見つけ、共有ストレージとアプリ固有ディレクトリの間で移動できます。

`rclone` も利用できるため、sync jobs の設定やクラウドストレージプロバイダー間のデータ移動が必要な場合は、やりたいことを説明すれば OpenClaw が設定を処理できます。

### 監視と診断

サービス監視に Uptime Kuma をインストールしている場合、OpenClaw は監視環境の上に会話レイヤーを提供します。単にアラートを受け取るだけでなく、OpenClaw にネットワーク診断を実行させられます。`ping`、`dig`、`nslookup`、`traceroute`、`mtr` に加え、より深い調査用に `nmap` と `tcpdump` まで含まれています。

何かが落ちましたか？アシスタントにメッセージを送り、何が起きているか確認させれば、ダッシュボードをにらみ続ける代わりに、平易な言葉で説明を受け取れます。

### 音楽とオーディオブックライブラリ

Navidrome と audiobookshelf のユーザーは、OpenClaw のメディアツールを活用できます。音楽ライブラリをスキャンして不足・誤った metadata tags を持つファイルを探したり、再エンコードが必要そうなトラックを特定したり、正しく分割されていないオーディオブック章の整理を手伝わせたりできます。`ffmpeg` と `exiftool` の組み合わせで、必要なことの多くを処理できます。

### Jupyter とデータ作業

データ分析に Jupyter Notebook を使っているなら、OpenClaw はよい補完になります。Python 3 と pip、venv サポートを備えているため、スクリプト作成、共有ストレージ内のデータファイル処理、簡単な分析の直接実行まで手伝えます。ノートブックが扱っている同じデータへの会話型インターフェイスだと考えてください。

### Infrastructure as Code

より技術的に踏み込む方向けに、OpenClaw には `docker`、`kubectl`、`terraform`、`ansible` CLI が含まれています。Appbox 上で VPS を実行しインフラを管理している場合、OpenClaw を会話型 ops assistant として使えます。デプロイしたいものを説明し、設定の生成と適用を手伝わせることができます。

## 永続メモリ

OpenClaw が単なる AI モデルへのチャットウィンドウと違う点の1つは、あなたを記憶することです。時間をかけて文脈を蓄積します。あなたの好み、整理方法、取り組んでいるプロジェクト、使っている命名規則。やり取りすればするほど、より役に立つようになります。

これは Appbox 上では特に強力です。文脈が会話だけではないからです。実際の環境、つまりファイル、アプリ、データに根ざしています。

## プリロード済みスキル

OpenClaw にはスキルシステムがあり、[ClawHub](https://clawhub.ai/skills) で50以上のコミュニティ製 extensions を利用できます。スキルは、特定のアプリ、ツール、ワークフロー向けの専門能力を agent に与えます。

Appbox では、bare agent だけを提供するわけではありません。イメージには厳選されたスキルセットがすでにインストールされており、OpenClaw は最初からあなたの環境を理解できます。

- **Platform awareness**: Appbox の共有データレイアウト、利用可能な CLI ツール、必要に応じたサポートや VPS アップグレードへの案内方法を理解しています
- **App-specific skills**: WordPress、Nginx、GitLab、Nextcloud、Emby、MariaDB、Tautulli、Mautic、WireGuard、Rocket.Chat 向けにプリロードされており、これらのアプリがインストールされている場合、agent はすでに扱い方を知っています
- **SSH essentials**: agent から VPS やリモートサーバーへ直接接続するためのものです
- **Browser automation (Playwright MCP)**: 他のアプリの Web UI にログインして操作する能力を支えます
- **Web search and documentation lookup**: 参照資料をその場で取り込むための multi-engine search と ClawDDocs
- **Self-improvement**: ClawHub 上の新しいスキルを発見し、インストール前に評価するためのスキル

追加スキルはいつでもインストールできます。[ClawHub](https://clawhub.ai/skills) で見つけ、リンクをコピーして agent に貼り付けるだけです。agent が自分でスキルをダウンロードしてインストールします。

## はじめ方

他のアプリと同じように、Appbox App Store から OpenClaw をインストールしてください。2スロットを使用します。

インストール後、アプリ詳細ページに Gateway Token が表示されます。これがダッシュボードログインです。ダッシュボード URL を開き、token を入力すれば入れます。

そこから設定が必要なのは AI model provider だけです。好みのプロバイダーの API キーを追加してください。OpenRouter は1つのキーで多くのモデルにアクセスできるため、最も簡単な出発点です。OpenRouter、OpenAI、Anthropic、Google、xAI、Mistral などから API キーを取得する手順を案内する [full setup guide](/docs/apps/openclaw-model-provider-setup) を用意しています。

モデル設定が完了したら、ダッシュボードからすぐにアシスタントとチャットできます。WhatsApp、Telegram、Discord、その他のチャネルから話しかけたい場合も、ダッシュボードで設定できます。

## なぜ重要なのか

AI チャットインターフェイスは世の中にたくさんあります。その多くはクラウドサービスで、メッセージを送った瞬間にデータが自分の管理下を離れます。OpenClaw は違います。すべてがあなたの Appbox 上に残ります。アシスタント、そのメモリ、会話、そして扱うデータもです。

さらに、共有ファイルシステムアクセスを持って他のアプリの隣にいるため、self-hosted stack 全体について質問に答えるだけでなく、実際に作業できます。

これで皆さんが何を作るのか、本当に楽しみにしています。面白いワークフローを設定したり、OpenClaw を他のアプリとつなぐ創造的な方法を見つけたりしたら、ぜひ教えてください。
