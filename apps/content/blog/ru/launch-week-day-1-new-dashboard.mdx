## Пользовательские домены стали полноценной частью продукта

Пользовательские домены теперь являются основной частью установки, а не неудобным дополнительным шагом.

Когда приложению нужен домен, настройка теперь происходит прямо в диалоге установки с понятным пошаговым процессом:

1. Выберите свой Appbox.
2. Выберите **Domain Type**:
   - **Appbox Domain**, чтобы использовать домен Appbox по умолчанию.
   - **Custom Domain**, чтобы использовать один из своих доменов.
3. Выберите **Base Domain**.
4. Введите нужный **Subdomain**.
5. Если вы выбрали пользовательский домен, Appbox покажет точную DNS-запись, которую нужно добавить (host/name и target IP), а затем позволит сразу проверить ее.

Вместо ручного угадывания записей, копирования документации и повторных попыток установки вы получаете встроенную настройку DNS и проверку там же, где устанавливаете приложение.

## Новый File Explorer (создан с нуля)

File Explorer был переосмыслен с нуля. В прежнем опыте управление файлами ограничивалось удалением. Теперь это полноценный файловый менеджер с:

- загрузками
- созданием новых папок
- переименованием
- удалением
- скачиванием
- переключением списка/сетки
- breadcrumb navigation и контекстными действиями

Теперь вы действительно можете управлять файлами appbox нативно, не устанавливая дополнительное приложение.

Безопасность не должна быть опциональной в 2026 году. Начиная с сегодняшнего дня аккаунты Appbox поддерживают встроенную двухфакторную аутентификацию.

Вы можете включать и управлять 2FA прямо из раздела аккаунта, включая управление recovery codes для восстановления доступа.
