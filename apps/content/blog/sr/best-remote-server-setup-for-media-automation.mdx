# Најбоља remote server поставка за media automation

Добра media automation поставка треба да делује досадно на најбољи могући начин. Ваше апликације остају online, датотеке су лако доступне, библиотека остаје организована и не обнављате proxy правила или storage mount-ове сваки пут када додате још један алат.

Постоје два уобичајена начина да изградите такву удаљену поставку:

- Почните са празним VPS-ом и сами склопите цео stack.
- Почните са хостованим апликацијама, shared storage-ом и контролама у dashboard-у, па додајте VPS само када вам треба custom контрола на нивоу машине.

Оба могу да раде. За већину људи, други пут је лакше одржати здравим током времена.

## Почните од workflow-а

Пре него што изаберете сервер, мапирајте workflow који заправо желите:

- Media server за репродукцију и преглед библиотеке.
- Monitoring да видите stream-ове, историју, bandwidth и активност сервера.
- Shared storage који све повезане апликације могу да користе.
- Приступ датотекама за upload-е, премештања, преименовања и чишћење.
- Sync или backup алати за чување важних датотека.
- Опциони custom scripts за послове који се не уклапају у app једним кликом.

Та листа је важна јер media automation ретко представља једну апликацију. Обично је то мали систем апликација које морају поуздано да деле датотеке.

## Прво изаберите media server

Media server је центар поставке.

[Plex](/apps/Plex) је исполирана опција са широком подршком клијената и познатим искуством библиотеке. [Jellyfin](/apps/Jellyfin) је потпуно open-source опција без subscription lock-in-а. [Emby](/apps/Emby) је у сличној personal-media-server категорији са сопственим client ecosystem-ом и моделом управљања.

## Рано додајте monitoring

Monitoring није само за велике поставке. Он вам помаже да разумете да ли је репродукција direct play или транскодирање, који корисници су активни и да ли се сервер понаша онако како очекујете.

За Plex поставке, [Tautulli](/apps/Tautulli) је очигледна companion app. Даје вам историју репродукције, stream детаље, активност корисника, видљивост bandwidth-а и event-based обавештења.

Та врста видљивости помаже када подешавате поставку. Можете да видите да ли неки уређај форсира непотребно транскодирање, да ли скенирање библиотеке прави оптерећење или да ли се удаљени stream мучи.

## Нека приступ датотекама буде једноставан

File layer је место где многи самостално изграђени media stack-ови постају неуредни.

Желите поставку у којој media server, management tools и file access алати разумеју исто складиште. Ако свака апликација има сопствену изоловану путању или custom mount, систем постаје тежи за разумевање.

На Appbox-у хостоване апликације могу да деле исто Appbox data окружење. Можете да користите file-focused апликације из библиотеке, као што су [File Browser](/apps/File%20Browser%20%26%20Webdav), [SFTPGo](/apps/SFTPGo), [Rclone](/apps/Rclone), [Syncthing](/apps/Syncthing) или [Nextcloud](/apps/Nextcloud), у зависности од тога да ли вам треба browser управљање датотекама, protocol access, cloud sync или лично cloud складиште.

## Додајте VPS само тамо где помаже

VPS је користан када вам треба пуна Linux контрола: custom scripts, инсталација пакета, scheduled jobs, development tools или комбинације апликација које су превише специфичне за app једним кликом.

Не морате да ставите целу media поставку на VPS само зато што један део workflow-а захтева ту контролу.

Са Appbox-ом, [Ubuntu VPS](/apps/Ubuntu%20VPS) може да приступи вашим Appbox подацима под `/APPBOX_DATA/`, укључујући податке хостованих апликација и shared storage. То значи да ваше хостоване апликације могу да остану у управљаном app flow-у, док VPS обавља custom automation над истим датотекама.

На пример:

- Покрећите Plex, Jellyfin, Emby или Tautulli као хостоване апликације.
- Користите File Browser, SFTPGo, Rclone, Syncthing или Nextcloud за приступ датотекама и sync.
- Додајте Ubuntu VPS за custom scripts, batch jobs, development tools или maintenance workflows.
- Држите све усмерено ка истим Appbox подацима уместо да делите поставку преко неповезаних сервера.

## Шта избегавати

Избегавајте почетак са празним сервером осим ако заиста желите сами да управљате оперативним системом, reverse proxy-јем, SSL сертификатима, firewall-ом, service files, storage mount-овима, ажурирањем пакета и troubleshooting-ом.

Избегавајте инсталирање сваке апликације одједном. Мања поставка је лакша за разумевање, подешавање и опоравак.

Избегавајте да сами од себе сакријете приступ датотекама. Ако не можете брзо да видите где живе app data и media датотеке, будуће одржавање ће бити спорије него што мора.

Избегавајте избор алата само по feature list-ама. Најбоља поставка је она којом и после три месеца можете самоуверено да управљате.

## Практичан Appbox образац

За већину remote media automation поставки, ово је добар почетни образац:

1. Инсталирајте омиљени media server из [Appbox app библиотеке](/apps).
2. Додајте [Tautulli](/apps/Tautulli) ако покрећете Plex и желите monitoring.
3. Додајте file-access апликацију као што су [File Browser](/apps/File%20Browser%20%26%20Webdav), [SFTPGo](/apps/SFTPGo) или [Nextcloud](/apps/Nextcloud).
4. Додајте [Rclone](/apps/Rclone) или [Syncthing](/apps/Syncthing) ако вашем workflow-у треба sync, migration или backup-style премештање датотека.
5. Додајте [Ubuntu VPS](/apps/Ubuntu%20VPS) само када вам требају custom jobs или потпуна контрола машине.

То вам даје корисне делове remote server-а без претварања сваког задатка у server-admin пројекат.

## Најбоља поставка је она коју можете да одржавате

Media automation треба да смањи трење, а не да створи други посао.

Ако уживате у ручној изградњи сваког слоја, VPS може бити прави алат. Ако вам је циљ да покрећете media апликације, надгледате их, управљате датотекама и одржавате систем online, хостоване апликације плус shared storage су обично боља почетна тачка.

Почните од app workflow-а. Додајте пуну VPS контролу само тамо где јасно заслужује место.
