# Launch Week дан 3: виртуелни приватни сервери

Дан 3. Ово је оно велико.

Од самог почетка, Appbox је био усмерен на то да self-hosted апликације учини једноставним. Инсталирате апликацију, добијете URL, готово. Тај модел функционише невероватно добро за већину ствари. Али увек је постојала класа случајева употребе у којима вам једноставно треба цела машина. Ваш сопствени kernel, ваш сопствени init систем, ваша сопствена правила.

Данас покрећемо **Virtual Private Servers** на Appbox-у.

## Праве VM, не контејнери

Да будемо јасни шта је ово. Ово су потпуне KVM виртуелне машине које раде на наменском хардверу, са сопственим kernel-ом, сопственим systemd-ом и пуним root приступом. Ово није Docker контејнер који се претвара да је OS. Ово је права ствар.

Та разлика је важна јер откључава све што контејнери не могу:

- **Docker унутар вашег VPS-а** - покрените `docker compose up` као на било ком серверу
- **Snaps и Flatpaks** - инсталирајте desktop software онако како је предвиђено
- **Systemd services** - потпуно управљање сервисима, timers, journal, све
- **Custom kernels and modules** - имате root, радите шта желите
- **Nested virtualization** - ако вам треба, ту је

Ако сте икада ударили у зид са нашим Docker-based OS-евима или webtop-овима где нешто једноставно није радило због ограничења контејнера, тај зид је нестао.

## Уграђена web конзола

Сваки VPS долази са browser-based конзолом уграђеном директно у Appbox контролну таблу. Кликните на Console tab на било ком VPS-у и добијате live terminal сесију директно у browser-у.

То значи да никада не можете да останете закључани ван свог сервера. Погрешно конфигурисан SSH? Покварена network config? Није важно. Web конзола се повезује директно на virtual display VM-а, потпуно независно од guest network stack-а. Увек имате начин да уђете.

## Ubuntu Desktop са Selkies

Овде постаје занимљиво. Наш Ubuntu VPS има desktop варијанту коју покреће **Selkies**, што вам даје потпуно графичко Linux desktop окружење доступно из browser-а, са hardware-accelerated streaming-ом, пуним audio output-ом и microphone input-ом.

Радили смо директно са Selkies тимом да ово покренемо унутар KVM виртуелних машина, а интеграцију смо објавили као open-source на [github.com/appbox-co/vm-webtop](https://github.com/appbox-co/vm-webtop).

Шта то заправо значи у пракси? Можете да покрећете пуно desktop окружење у browser-у са правим апликацијама. VS Code, Discord, Spotify, Slack - све ради истовремено, све са звуком, све доступно са било ког уређаја који има web browser.

## Ваши подаци иду са вама

Сваки VPS има аутоматски приступ вашем Appbox складишту преко директоријума `/APPBOX_DATA/`. Унутра ћете наћи:

- **`/APPBOX_DATA/apps/`** - подаци из других инсталираних Appbox апликација
- **`/APPBOX_DATA/storage/`** - ваше опште Appbox складиште

Ово се дели помоћу virtiofs, high-performance filesystem passthrough-а који даје брзине приступа датотекама блиске native приступу. Ваш VPS може да чита и пише датотеке које користе и ваше друге апликације, тако да, на пример, можете да обрађујете media датотеке из streaming апликација или правите backup база података из свог VPS-а.

## Ради са App Boost-ом

VPS-еви користе исти App Boost систем који смо покренули другог дана. Желите више RAM-а или CPU-а за своју VM? Померите boost ниво нагоре и посветите јој више slot-ова из свог плана. Ресурси се алоцирају на нивоу hypervisor-а, тако да ваша VM заиста добија више compute ресурса, а не само виши cgroup лимит.

Вреди напоменути: VPS-еви имају мало виши основни slot cost од Docker-based OS апликација. То одражава чињеницу да потпуна виртуелна машина троши више стварних ресурса него контејнер који дели host kernel. Компромис је једноставан: добијате више могућности, а slot cost то искрено одражава.

## Шта следи за VPS

Данас покрећемо VPS-еве са пуном функционалношћу, али има још тога на roadmap-у. Графикони употребе ресурса за VPS инстанце биће додати у блиској будућности, тако да ћете моћи да пратите CPU, RAM и disk usage током времена директно из контролне табле, баш као што можете са другим апликацијама.

## Зашто је ово важно

Docker-based OS-еви и webtop-ови на Appbox-у добро су служили многим људима и не иду нигде. Али постоје стварна ограничења рада унутар контејнера. Не можете да покрећете Docker. Не можете да користите snaps. Не можете да учитавате kernel modules. Не можете да покренете потпун init system. Увек радите у оквирима туђег namespace-а.

VPS-еви у потпуности уклањају та ограничења. Добијате праву машину. Ако нешто можете да урадите на серверу изнајмљеном код било ког другог провајдера, можете то да урадите и овде, с тим што су ваши подаци већ монтирани, web конзола је већ ту, а ресурси се скалирају slider-ом.

## Дан 3 је сада доступан

VPS-еви су доступни у App Store-у одмах. Идите у VPS категорију, изаберите OS и инсталирајте. Provisioning пуне VM траје мало дуже од container app-а, па очекујте до 15 минута да ваш VPS буде спреман.
