# Launch Week дан 4: више домена за VPS апликације

Дан 4. Јуче смо вам дали потпуне виртуелне машине. Данас им дајемо праву подршку за домене.

Када смо покренули VPS-еве трећег дана, свака VM је добила један домен. То је добро за много случајева употребе, али ако покрећете web server унутар свог VPS-а, а многи од вас то раде, један домен није довољан. Потребно вам је више домена који показују на исту машину, сваки са сопственим SSL сертификатом и конфигурацијом web server-а.

Данас VPS апликације подржавају **више домена**. Додајте онолико `*.user.appboxes.co` поддомена колико желите, донесите сопствене прилагођене домене или комбинујте оба. Сваки добија SSL и спремну web server конфигурацију.

## Како функционише

Отворите било коју покренуту VPS апликацију и на страници са детаљима наћи ћете **Custom Domains** картицу. Ту живе сви ваши домени: searchable, sortable табела која приказује сваки домен повезан са тим VPS-ом, који је primary и када је додат.

Да додате нови домен, кликните **Add Domain**. Изабраћете parent domain и унети subdomain, а систем се брине за остало.

## Appbox поддомени

Најбржи начин да додате домен је да користите свој постојећи Appbox домен. Изаберите свој `user.appboxes.co` домен као parent, унесите subdomain и готово. DNS је већ решен за вас, а домен се верификује одмах.

Ово је одлично за брзо подизање project URL-ова, staging окружења или једноставно да различитим сервисима на свом VPS-у дате сопствену адресу. Треба вам `api.user.appboxes.co` за backend и `app.user.appboxes.co` за frontend? Додајте оба за мање од минут.

  <img src="/blog-images/day4-add-subdomain.png" alt="Додавање Appbox поддомена на VPS" width="560" className="mt-8 mb-10" />

Пошто се налазе испод вашег постојећег Appbox wildcard сертификата, SSL је аутоматски. Без чекања, без генерисања сертификата, без конфигурације.

## Прилагођени домени

Такође можете да усмерите сопствене домене на свој VPS. Изаберите свој custom base domain (или додајте нови директно из дијалога), унесите subdomain, а UI вас води кроз DNS подешавање.

Дијалог приказује тачан A record који треба да креирате: hostname, IP адресу, све. Чак има и provider-specific упутства за Cloudflare и друге популарне DNS провајдере, тако да не морате да тражите како се додаје A record.

Када ваш DNS показује на праву IP адресу, притисните **Verify Again** и систем потврђује да је запис исправан пре додавања домена.

## SSL и web server конфигурација

Сваки домен који додате добија SSL. За Appbox поддомене, користи се постојећи wildcard сертификат. За прилагођене домене, сертификат се provisioning-ује аутоматски.

Ево дела који ово чини заиста корисним за VPS кориснике: сваки домен долази са **View Config** дугметом које вам даје configuration snippets спремне за copy-paste и за Nginx и за Apache. Config укључује исправан `server_name`, путање SSL сертификата, cipher suites и protocol settings. Копирајте snippet, убаците га у web server config унутар VM-а, reload-ујте и служите саобраћај на том домену.

## Зашто је ово важно за VPS кориснике

VPS без подршке за више домена је само сервер са једном адресом. То је ограничење које брзо постаје очигледно:

- Покрећете web app са одвојеним API-јем? Требају вам најмање два домена.
- Хостујете више сајтова у својој VM? Сваком треба сопствени домен.
- Желите staging окружење поред production-а? То је још један домен.
- Развијате локално и желите да тестирате са правим доменом? Још један.

## Дан 4 је сада доступан

Подршка за више домена доступна је одмах за све VPS апликације. Отворите свој VPS, скролујте до Custom Domains и почните да додајете.
