# Приватне cloud апликације за media и downloads

Приватна cloud поставка је најкориснија када доведе ваше датотеке, media садржај, downloads и sync алате на једно место.

То не мора да значи изградњу сложеног server stack-а од нуле. Боља почетна тачка је обично скуп фокусираних апликација које чисто деле складиште: једна апликација за file sync, једна за browser приступ датотекама, једна за media playback, једна за transfers или backup-е и опциони додатни алати када workflow порасте.

## Шта је потребно приватној cloud поставци

Тачна листа апликација зависи од вашег workflow-а, али већини приватних cloud поставки треба неколико слојева:

- File sync и дељење.
- Browser-based управљање датотекама.
- Безбедан file transfer приступ.
- Media library репродукција.
- Организација фотографија, видеа, музике или докумената.
- Backup, migration или scheduled transfer алати.
- Опциона контрола на нивоу машине за custom scripts.

Корисно код Appbox-а је то што ово могу бити одвојене апликације без претварања у одвојена острва. Хостоване апликације могу да деле исто Appbox storage окружење, а Appbox VPS може да приступи истим подацима преко `/APPBOX_DATA/` када вам треба пуна Linux контрола.

## Почните са file sync-ом

[Nextcloud](/apps/Nextcloud) је природна почетна тачка за многе приватне cloud поставке. Даје вам file sync клијенте, контроле дељења, WebDAV приступ и познат cloud-drive workflow.

Користите га када желите:

- Приватно складиште датотека са desktop и mobile sync-ом.
- Share links са лозинкама или датумима истека.
- Централно место за документе, project files и personal media.
- WebDAV компатибилност за апликације и desktop клијенте.

Nextcloud није једина file апликација коју можете користити, али често је она због које поставка делује као personal cloud, а не као лабава колекција алата.

## Додајте browser приступ датотекама

File sync је користан, али понекад само треба да отворите browser и преместите датотеке.

[File Browser](/apps/File%20Browser%20%26%20Webdav) и [File Browser Quantum & WebDAV](/apps/File%20Browser%20Quantum%20%26%20Webdav) покривају ту једноставну потребу за управљањем датотекама. Корисни су за upload-е, download-е, преименовања, чишћење folder-а и брзе провере са машине на којој не желите да конфигуришете пун sync клијент.

Тај browser layer постаје посебно користан када media и download folder-и порасту. Можете да прегледате структуру датотека, преместите завршене датотеке у library folder-е или средите привремено складиште без отварања terminal-а.

## Додајте transfer алате за веће workflow-ове

Приватним cloud поставкама често треба више од ручних upload и download дугмади.

[SFTPGo](/apps/SFTPGo) је снажан избор када вам требају SFTP, FTP/S, HTTP/S, WebDAV, user controls, virtual folders или policy-based access. Користан је када желите приступ датотекама на нивоу протокола, а не само web UI.

[Rclone](/apps/Rclone) је користан за copy, sync, migration и backup-style workflow-ове између cloud storage провајдера и remote filesystems.

[Syncthing](/apps/Syncthing) и [Resilio Sync](/apps/Resilio%20Sync) су корисни када желите device-to-device sync без третирања third-party cloud drive-а као центра система.

За download workflow-ове, држите дизајн једноставним и легитимним: складиштите датотеке које имате право да складиштите и преносите, држите привремене folder-е одвојене од library folder-а и користите sync или backup алате намерно, уместо да свакој апликацији дозволите да пише свуда.

## Додајте media апликације око истог складишта

Када су ваше датотеке организоване, media апликације чине приватни cloud кориснијим.

[Plex](/apps/Plex), [Jellyfin](/apps/Jellyfin) и [Emby](/apps/Emby) су главне media-server опције за video, music и photo библиотеке. [Tautulli](/apps/Tautulli) додаје monitoring за Plex активност, stream детаље и употребу библиотеке.

За фокусираније media библиотеке, [Immich](/apps/Immich) је користан за приватно управљање фотографијама и видео садржајем, [Navidrome](/apps/Navidrome) је користан за music streaming, [audiobookshelf](/apps/audiobookshelf) је користан за audiobooks и podcasts, а [Calibre](/apps/Calibre) је користан за ebook библиотеке.

Важно је да media апликације показују на storage layout који разумете. Приватну cloud поставку је лакше одржавати када downloads, imports, libraries и backups имају јасне folder-е и ownership.

## Држите downloads одвојено од libraries

Једна једноставна навика спречава много чишћења касније: држите incoming датотеке одвојене од завршених libraries.

На пример:

- `/downloads/incoming/` за нове датотеке.
- `/downloads/processing/` за датотеке које се проверавају, преименују или сортирају.
- `/media/video/`, `/media/music/`, `/media/photos/` или `/media/books/` за завршене libraries.
- `/backups/` за snapshots, exports или off-site copies.

То раздвајање чини аутоматизацију безбеднијом. Sync job може да копира само завршене датотеке. Media server може да скенира само library folder-е. Cleanup task може да уклони привремене датотеке без додиривања важних података.

## Када VPS помаже

Хостоване апликације покривају уобичајени workflow. VPS помаже када вам треба нешто custom:

- Scheduled scripts.
- Custom file-processing jobs.
- Development tools.
- One-off migrations.
- Package installs или command-line utilities.
- Аутоматизација која се не уклапа у app template.

[Ubuntu VPS](/apps/Ubuntu%20VPS) на Appbox-у може да приступи Appbox подацима под `/APPBOX_DATA/`, тако да може да ради са истим датотекама које користе ваше хостоване апликације. То значи да можете да држите Nextcloud, File Browser, Plex, Jellyfin или Immich у app flow-у, а да користите VPS за необичне послове којима треба пуна контрола машине.

Овај hybrid модел смо детаљније покрили у [Хостовани app server или VPS](/blog/sr/hosted-app-server-vs-vps).

## Практичан app stack

За уравнотежену приватну cloud поставку, почните мало:

1. [Nextcloud](/apps/Nextcloud) за sync, дељење и лично cloud складиште.
2. [File Browser](/apps/File%20Browser%20%26%20Webdav) или [File Browser Quantum & WebDAV](/apps/File%20Browser%20Quantum%20%26%20Webdav) за browser управљање датотекама.
3. [SFTPGo](/apps/SFTPGo), [Rclone](/apps/Rclone), [Syncthing](/apps/Syncthing) или [Resilio Sync](/apps/Resilio%20Sync) за transfers, migration, backup или device sync.
4. [Plex](/apps/Plex), [Jellyfin](/apps/Jellyfin) или [Emby](/apps/Emby) за media playback.
5. [Immich](/apps/Immich), [Navidrome](/apps/Navidrome), [audiobookshelf](/apps/audiobookshelf) или [Calibre](/apps/Calibre) ако ваша библиотека има специфичан облик.
6. [Ubuntu VPS](/apps/Ubuntu%20VPS) само када custom automation то захтева.

То је довољно да изградите приватни cloud који покрива датотеке, media садржај, downloads и remote access без претварања сваког корака у ручно одржавање сервера.

## Градите за одржавање

Најбоља приватна cloud app поставка није она са најдужом листом апликација. То је она коју можете да разумете, користите и опоравите.

Почните са малим app stack-ом, држите границе folder-а јасним и додајте алате само када решавају стварни workflow проблем. Appbox вам даје app библиотеку, shared storage модел и опциону VPS контролу, тако да поставку можете да развијате без поновне изградње темеља сваки пут.
