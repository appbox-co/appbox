# เราทำสำเร็จแล้ว - Network ของเรา Live แล้ว!

จำได้ไหมเมื่อเดือน April ที่เราประกาศว่าเรากำลังสร้าง network infrastructure ของเราเอง? ตอนนี้ผมตื่นเต้นที่จะบอกคุณว่า ณ เมื่อวานนี้ มัน live แล้วในที่สุด

นี่เป็นงานที่ใช้เวลาหลายเดือน และพูดตรง ๆ การได้เห็นทุกอย่างประกอบเข้าด้วยกันเป็นหนึ่งในช่วงเวลาที่น่าพอใจที่สุดตั้งแต่เราเริ่ม Appbox มา ให้ผมเล่าให้ฟังว่ามีอะไรเปลี่ยนไปบ้าง

## Hardware ระดับองค์กร

เรา deploy Nokia 7750 SR-1 routers ทั่ว infrastructure ของเรา หากคุณไม่คุ้นเคยกับ routing gear ของ Nokia นี่คือ routers แบบเดียวกับที่ใช้ขับเคลื่อน ISP และ data centers ที่ใหญ่ที่สุดบางแห่งในโลก มันถูกสร้างมาเพื่อ reliability และ performance ในระดับ scale และตอนนี้กำลังจัดการ traffic ทั้งหมดของคุณ

นี่ไม่ใช่อุปกรณ์ consumer หรือ prosumer นี่คือ enterprise networking hardware ที่เหมาะสม ซึ่งให้ control และ capabilities ที่เราต้องการเพื่อส่งมอบ performance ที่คุณสมควรได้รับ

## IP Space ของเราเอง

ตอนนี้เรากำลัง route traffic ผ่าน IP addresses ของเราเองภายใต้ Autonomous System ของเรา ([AS209870](https://apps.db.ripe.net/db-web-ui/lookup?source=RIPE&type=aut-num&key=AS209870)) นี่หมายความว่าเราควบคุมได้ทั้งหมดว่าข้อมูลของคุณเดินทางผ่าน internet อย่างไร

ทำไมสิ่งนี้สำคัญ? เพราะเราสามารถ optimize routing เฉพาะสำหรับผู้ใช้ Appbox, implement DDoS protection ที่ดีขึ้น และในอนาคตเสนอ features อย่าง dedicated IP addresses สำหรับผู้ที่ต้องการ

## 100G AMS-IX Connectivity

นี่คือเรื่องใหญ่ ตอนนี้เรา peering กับ AMS-IX ที่ 100 gigabits per second AMS-IX เป็นหนึ่งใน internet exchanges ที่ใหญ่ที่สุดในโลก และการมี direct access หมายความว่า traffic ของคุณใช้เส้นทางที่เร็วที่สุดเท่าที่เป็นไปได้ไปยัง major networks และ content providers

ในแง่จริง สิ่งนี้หมายความว่า:
- **Latency ต่ำลง** - connections ของคุณเร็วขึ้นและตอบสนองดีขึ้น
- **Speed ดีขึ้น** - bandwidth มากขึ้นหมายถึง congestion น้อยลง
- **Reliability ดีขึ้น** - direct peering หมายถึงจุดล้มเหลวน้อยลง

เราได้ทดสอบสิ่งนี้ในช่วงหลายสัปดาห์ที่ผ่านมากับ traffic บางส่วน และผลลัพธ์ตรงตามที่เราหวังไว้ ตอนนี้ทุกคนจะได้ประโยชน์

## ต่อไปคืออะไร

network upgrade นี้เป็นเพียง foundation ตอนนี้เมื่อเราควบคุม infrastructure ของเราได้อย่างเหมาะสม เราสามารถเริ่มสร้าง features ที่ก่อนหน้านี้เป็นไปไม่ได้:

- **Dedicated IP addresses** - สำหรับผู้ใช้ที่ต้องการ IP ของตัวเอง
- **Advanced routing options** - ควบคุมวิธีที่ traffic ของคุณไหลมากขึ้น
- **Enhanced DDoS protection** - ใช้ infrastructure ใหม่ของเราเพื่อความปลอดภัยที่ดีขึ้น
- **Geographic optimization** - เมื่อเราขยาย peering relationships

เราจะอัปเดตให้คุณทราบเมื่อ features เหล่านี้พัฒนาต่อไป
