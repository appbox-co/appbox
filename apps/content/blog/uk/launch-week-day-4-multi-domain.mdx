# Launch Week, день 4: кілька доменів для VPS apps

День 4. Учора ми дали вам повноцінні virtual machines. Сьогодні даємо їм належну підтримку доменів.

Коли ми запустили VPS у день 3, кожна VM отримувала один домен. Для багатьох сценаріїв цього достатньо, але якщо ви запускаєте web server усередині свого VPS - а багато хто з вас це робить - одного домену замало. Потрібні кілька доменів, що вказують на ту саму машину, кожен зі своїм SSL certificate і конфігурацією web server.

Сьогодні VPS apps підтримують **кілька доменів**. Додавайте скільки завгодно subdomains `*.user.appboxes.co`, підключайте власні custom domains або поєднуйте обидва варіанти. Кожен отримує SSL і готову до використання конфігурацію web server.

## Як це працює

Відкрийте будь-який запущений VPS app, і на сторінці деталей побачите картку **Custom Domains**. Саме тут живуть усі ваші домени - таблиця з пошуком і сортуванням, яка показує кожен домен, прив'язаний до цього VPS, який із них primary і коли його додано.

  <img src="/blog-images/day4-domains-table.png" alt="Таблиця Custom Domains на сторінці деталей VPS app" width="860" className="mt-8 mb-10" />

Щоб додати новий домен, натисніть **Add Domain**. Ви виберете батьківський домен і введете subdomain, а система подбає про решту.

## Subdomains Appbox

Найшвидший спосіб додати домен - використати ваш наявний домен Appbox. Виберіть свій домен `user.appboxes.co` як батьківський, введіть subdomain, і все готово. DNS уже обробляється за вас, а домен перевіряється миттєво.

Це чудово підходить для швидкого запуску URL для проєктів, staging-середовищ або просто для того, щоб дати різним сервісам на вашому VPS власну адресу. Потрібен `api.user.appboxes.co` для backend і `app.user.appboxes.co` для frontend? Додайте обидва менш ніж за хвилину.

<p align="center">
  <img src="/blog-images/day4-add-subdomain.png" alt="Додавання subdomain Appbox до VPS" width="560" className="mt-8 mb-10" />
</p>

Оскільки вони розміщені під вашим наявним wildcard certificate Appbox, SSL працює автоматично. Без очікування, без генерації сертифікатів, без налаштувань.

## Користувацькі домени

Ви також можете спрямувати власні домени на свій VPS. Виберіть свій custom base domain (або додайте новий прямо з діалогу), введіть subdomain, і інтерфейс проведе вас через налаштування DNS.

Діалог показує точний A record, який потрібно створити, - hostname, IP address, усе. Він навіть має інструкції для конкретних провайдерів, зокрема Cloudflare та інших популярних DNS providers, тож вам не доведеться шукати, як додати A record.

Коли ваш DNS вказує на правильну IP, натисніть **Verify Again**, і система підтвердить, що запис правильний, перш ніж додати домен.

## SSL і конфігурація web server

Кожен доданий домен отримує SSL. Для subdomains Appbox використовується наявний wildcard certificate. Для custom domains сертифікат випускається автоматично.

Ось частина, яка робить це справді корисним для користувачів VPS: кожен домен має кнопку **View Config**, що дає готові до вставлення фрагменти конфігурації для Nginx і Apache. Конфігурація містить правильний `server_name`, шляхи до SSL certificate, cipher suites і налаштування протоколів. Скопіюйте фрагмент, вставте його у конфігурацію web server всередині VM, перезавантажте - і ви вже обслуговуєте трафік на цьому домені.

Жодних здогадок щодо шляхів до файлів. Жодного полювання за розташуванням сертифікатів. Жодного розбирання cipher strings. Усе вже там, готове до копіювання.

## Чому це важливо для користувачів VPS

VPS без підтримки кількох доменів - це просто сервер з однією адресою. Це обмеження швидко стає очевидним:

- Запускаєте web app з окремим API? Потрібно щонайменше два домени.
- Хостите кілька сайтів у своїй VM? Кожному потрібен власний домен.
- Хочете staging-середовище поруч із production? Це ще один домен.
- Розробляєте локально й хочете тестувати зі справжнім доменом? Потрібен ще один.

## День 4 уже доступний

Підтримка кількох доменів доступна прямо зараз для всіх VPS apps. Відкрийте свій VPS, прокрутіть до Custom Domains і починайте додавати.
