## Windows в Appbox

Коли ми запустили VPS у день 3 із п'ятьма дистрибутивами Linux, природним запитанням було: "а як щодо Windows?" Змусити Windows належно працювати як керований VPS в Appbox було значним технічним викликом. Linux VMs використовують virtiofs для майже нативного спільного доступу до файлів між хостом і гостем. Windows не підтримує virtiofs. Тож ми побудували інтеграцію на базі Samba, яка створює автентифіковані SMB shares для кожної VM, даючи Windows такий самий безшовний доступ до ваших даних Appbox, який мають Linux VPS, - просто через протокол, який Windows розуміє нативно.

У результаті це повноцінна virtual machine Windows 11 Pro, що працює на KVM, із вашими даними Appbox, змонтованими як диск `Z:` (`APPBOX_DATA`). Відкрийте File Explorer, перейдіть до `Z:\`, і ви побачите свої папки `apps/` і `storage/` прямо там.

## Що це відкриває

Наявність Windows VPS в Appbox відкриває цілу нову категорію сценаріїв:

- **Software лише для Windows** - запускайте застосунки, які просто не мають Linux-версій
- **.NET development** - створюйте й тестуйте .NET-застосунки у справжньому Windows-середовищі
- **Офіс і продуктивність** - запускайте Microsoft Office, інструменти Adobe або будь-яке desktop software
- **Ігрові сервери** - хостіть Windows-only game servers
- **Віддалений desktop** - отримуйте доступ до повного Windows desktop з будь-якого браузера через вбудовану вебконсоль або підключайтеся через Microsoft Remote Desktop для нативного RDP-досвіду
- **Тестування та CI** - тестуйте своє програмне забезпечення на Windows без підтримки власного обладнання

Це повноцінна інсталяція Windows 11 Pro з доступом адміністратора. Якщо воно працює на Windows, воно працює тут.

## Доступність на планах

Windows 11 Pro має досить високі вимоги до ресурсів і створює значне навантаження на disk I/O. Щоб забезпечити хороший досвід для всіх, Windows VPS наразі доступний лише на виділених планах Appbox (NG-18000 і вище). У майбутньому ми можемо розширити доступність на інші плани, продовжуючи оптимізацію.

## Ліцензування

Windows 11 Pro VPS не містить product key. Windows працюватиме неактивованою з постійним водяним знаком на desktop, обмеженими налаштуваннями персоналізації та періодичними нагадуваннями про активацію, але в іншому буде повністю функціональною. Щоб прибрати ці обмеження, активуйте її дійсним ключем Windows 11 Pro. Ми рекомендуємо придбати його на [WinCDKey (~6 EUR)](https://wincdkey.com/product/microsoft-windows-11-professional-1-device-pc-software/).

## Працює з App Boost

Windows 11 Pro VPS повністю підтримує **App Boost** з дня 2. Якщо вам потрібна більша продуктивність для важчих навантажень, підніміть рівень boost, щоб виділити VM більше RAM і CPU.

## App Freeze

Сьогодні виходить і друга функція, яка стосується всіх застосунків, не лише Windows.

<p align="center">
  <img src="/blog-images/day5-windows-detail-freeze.png" alt="Сторінка деталей Windows 11 Pro VPS з App Boost і новою кнопкою Freeze" width="560" className="mt-8 mb-10" />
</p>

**App Freeze** дає змогу призупинити будь-який встановлений застосунок і повернути його app slots без втрати даних. Застосунок зупиняється, його дані залишаються саме там, де були, а слоти, які він використовував, стають доступними для інших застосунків.

Це розв'язує проблему, яка виникала постійно: ви хочете спробувати новий застосунок, але вже використали всі слоти. До сьогодні довелося б видалити щось - і втратити його дані - лише щоб звільнити достатньо слотів для експерименту. Тепер ви можете заморозити застосунок, використати ці слоти, щоб встановити й протестувати щось нове, а потім розморозити початковий застосунок, коли закінчите. Усе продовжиться саме з того місця, де ви зупинилися.

Кнопка Freeze розміщена прямо в секції Actions на сторінці деталей будь-якого застосунку, поруч із Start, Stop і Restart. Заморожування зупиняє застосунок, позначає його як frozen і повертає його слоти (включно з будь-якими boost slots) у ваш доступний пул. Розморожування відновлює застосунок і знову споживає слоти.

Кілька важливих моментів:

- Ви не можете заморозити застосунок, поки він встановлюється, оновлюється або видаляється
- Ви не можете розморозити застосунок, якщо у вашому Appbox недостатньо вільних слотів для покриття вартості
- Заморожені застосунки зберігають усі свої дані та конфігурацію без змін
