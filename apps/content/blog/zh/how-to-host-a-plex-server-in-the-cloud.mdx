# 如何在云端托管 Plex 服务器

在家运行 Plex 通常很好用，直到它不好用为止。上传速度、存储限制、硬件故障、硬盘噪音和远程访问设置，都可能让“打开媒体库然后播放”这件简单的事变得麻烦。

云端 Plex 服务器解决的是另一个问题：它为你提供一台始终在线、随处可访问的私有媒体服务器，而且不受你桌下那台电脑的限制。相应的取舍是，你会希望这套方案简单、可预期，并且围绕媒体应用构建，而不是从一台通用空白服务器开始。

本指南会说明选择云端 Plex 方案时真正重要的因素，以及 Appbox 如何融入这个工作流程。

## 云端 Plex 服务器需要什么

Plex 本身并不算复杂，但一个好用的托管 Plex 服务器，日常使用中仍然需要具备几项条件：

- **可靠的存储**，用于保存媒体库、元数据、海报和应用数据。
- **足够的 CPU 和内存**，用于媒体库扫描、远程访问和偶尔发生的转码。
- **快速的网络访问**，让远程播放能够快速开始并保持稳定。
- **固定 URL**，用于访问 Plex Web 应用和服务器。
- **简单的应用管理**，让更新、重启和凭据管理不至于变成周末工程。
- **运行配套应用的空间**，例如 Jellyfin、Emby、Tautulli、Overseerr、[File Browser](/apps/File%20Browser%20%26%20Webdav) 或 Nextcloud。

你当然可以自己在 VPS 上搭建这一切。很多人也确实这么做。但这也意味着你要负责操作系统、Docker 栈、防火墙、反向代理、SSL 证书、存储挂载、服务重启，以及随之而来的每一个运维细节。

Appbox 面向的是另一条路径：选择应用、安装应用，然后在仪表板里管理它。

## 为什么用 Appbox 托管 Plex

在 Appbox 上，Plex 是一个一键安装应用。你不是从一台裸服务器开始，而是从一个托管应用环境开始；这个环境已经处理好域名、应用数据、存储、资源分配和仪表板控制等周边能力。

这会带来一些很实际的好处：

- **从 Appbox 应用库一键安装**。
- **仪表板控制**，用于查看应用详情、重启、凭据、版本和资源用量。
- **Appbox 应用之间共享存储**，让相关工具可以处理同一批文件。
- **自定义域名和 HTTPS** 通过平台流程处理。
- **App Boost**，当较重的应用需要从你的套餐中获得更多 CPU 或内存时使用。
- **同一个应用库中的配套应用**，包括 [Jellyfin](/apps/Jellyfin)、[Emby](/apps/Emby)、[Tautulli](/apps/Tautulli)、[File Browser](/apps/File%20Browser%20%26%20Webdav) 和 [Nextcloud](/apps/Nextcloud)。

如果你的目标是运行 Plex，而不是成为自己媒体栈的运维团队，这些细节就很重要。

## Plex 托管与通用 VPS

VPS 很灵活。你拥有 root 访问权限，几乎可以安装任何东西。当你需要完全控制、自定义服务，或非常具体的 Linux 环境时，这很有用。

但对 Plex 托管来说，灵活性也可能变成阻力。典型的 VPS 搭建流程包括选择 Linux 发行版、安装 Docker 或软件包、配置存储、设置反向代理、管理 SSL、开放端口、创建备份，并自行监控资源使用情况。

Appbox 的取向更明确。它并不是想替代所有 VPS 使用场景，而是想让托管应用变得更简单。

如果你想自己管理每一层，请选择 VPS。

如果你想要一个以 Plex 和相关应用为主要工作流程的私有媒体应用环境，请选择 Appbox。

## 选择合适的 Appbox 套餐

合适的套餐取决于你还想在 Plex 旁边运行什么。

对于轻量设置，你可能只需要 Plex 加一个文件管理器。对于更完整的媒体环境，你可能会需要 Plex、Tautulli、File Browser、Nextcloud 以及几个辅助工具。如果你预计会有更重的扫描任务、更大的媒体库或多个活跃应用，一开始就要给这些需求留出足够的套餐容量。

Appbox 的实用之处在于，套餐不只是一个应用槽位，而是一个环境。你可以安装多个应用，在支持的情况下用 App Boost 分配资源，并让相关服务靠近同一份存储。

当你的媒体服务器从“只有 Plex”发展成一个小型私有应用栈时，这一点尤其有用。

## 在 Appbox 上设置 Plex

基本流程很简单：

1. 打开 [Plex 应用页面](/apps/Plex)。
2. 选择一个容量足够支撑你的媒体库和配套应用的 Appbox 套餐。
3. 通过应用流程安装 Plex。
4. 打开应用详情页面，找到生成的 URL 和凭据。
5. 登录 Plex，并把它连接到你的账号。
6. 添加媒体文件夹，让 Plex 扫描媒体库。

之后，你可以按需从 Appbox 应用库添加相关应用。

## 值得考虑的配套应用

Plex 往往是整套方案的中心，但它很少是唯一有用的应用。

[Tautulli](/apps/Tautulli) 适合用来监控 Plex 和媒体库活动。如果你想比较不同工作流程，或者想在 Plex 旁边运行开源方案，[Jellyfin](/apps/Jellyfin) 和 [Emby](/apps/Emby) 是可选的媒体服务器。[File Browser](/apps/File%20Browser%20%26%20Webdav) 提供直接好用的网页文件管理器。如果你希望在同一个 Appbox 环境中拥有私有云存储和文件同步，[Nextcloud](/apps/Nextcloud) 会很有用。

这也是 Appbox 不只是“Plex 主机”的地方。它是围绕你的媒体库和相关工具构建的托管应用环境。

## 几个实用建议

从一开始就保持媒体库结构整洁。Plex 很擅长匹配媒体，但清晰的文件夹和一致的命名会在以后节省很多时间。

尽可能使用直接播放。转码有时很有用，但它比直接串流设备已经支持的文件格式更吃资源。

留意存储增长。海报、元数据、缩略图和配套应用都会随着时间占用空间，不只是媒体文件本身会占空间。

让配套应用保持聚焦。一次安装所有东西很有诱惑力，但更小的设置更容易理解和调优。

## 从 Plex 应用页面开始

如果你想要一台私有 Plex 服务器，但不想手动搭建整个栈，可以从 [Plex 应用页面](/apps/Plex) 开始。你可以比较套餐、查看应用要求，并通过与其他应用库应用相同的 Appbox 流程完成安装。

如果以后想扩展到 Plex 之外，同一个环境也可以托管其他应用，让私有媒体服务器变得更有用。
