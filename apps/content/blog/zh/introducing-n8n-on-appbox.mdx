# Appbox 上线 n8n

n8n 现在已在 Appbox 上提供。

这意味着你可以部署一个私有的工作流自动化工作区，打开可视化编辑器，并开始连接应用、API、计划任务、webhook 和自定义代码，而无需手动管理服务器。

[从 Appbox App Store 安装 n8n](/appstore/app/264)。

## 你可以构建什么

当一个应用需要响应另一个应用、清理数据、调用 API、发送通知或运行计划任务时，n8n 都很有用。

在 Appbox 上，这让它天然适合搭配那些带有 webhook、API、通知钩子、WebDAV、RPC 或可脚本化接口的应用。

用于 AI、支持和业务工作流：

- **OpenClaw** 可以作为自动化中的 AI 层。当支持工单、监控告警或文件上传需要摘要时，从 n8n 发送 webhook，然后让 OpenClaw 起草回复、分类请求，或把事件转换成任务。
- **Chatwoot** 为客户支持暴露收件箱事件和 API。标记紧急对话，从另一个系统补全新联系人信息，在 VIP 发来消息时通知团队频道，或在对话关闭后创建跟进任务。
- **Mautic** 提供 REST API 和营销活动自动化能力。同步表单线索，在产品事件后为联系人评分，把 webhook 提交的人加入营销活动，或在线索超过阈值时通知销售团队。
- **ERPNext** 通过 API 暴露业务记录。根据表单提交创建发票，在付款到账时更新交易，在库存不足时通知运营，或把客户记录推送到另一个 Appbox 应用。
- **Plane** 和 **OpenProject** 很适合作为任务目标。把事件、表单提交、失败的作业或客户请求转换成问题和工作包，并带上正确的负责人和上下文。
- **Rallly** 可以融入日程安排工作流。根据请求表单创建投票，在会议时间选定后通知来宾，或在可用时间截止前发送提醒。
- **OnlyOffice** 和 **Jitsi** 适合文档与会议工作流。文档准备好后通知审阅者，审批通过后创建会议链接，或把会后跟进任务发送到 Plane 或 OpenProject。

用于内容、发布和开发者工作流：

- **EmDash** 提供带有表单、短链接、插件和自动构建钩子的结构化 CMS。将表单提交发送到 n8n，在内容发布时通知编辑，草稿上线后触发重新构建，或把新文章发送到 Mattermost、Rocket.Chat 和社交发布队列。
- **WordPress** 拥有 REST API 和钩子生态。根据表单提交起草文章，在内容上线时发布公告，把媒体备份到另一个存储应用，或在计划发布前发送编辑提醒。
- **GitLab**、**Gogs** 和其他 Git 应用会暴露仓库、项目和系统钩子。根据标签发送发布通知，合并后创建部署清单，把失败流水线摘要发送到聊天，或从缺陷报告模板创建项目管理问题。
- **Hugo Git** 可以位于基于 Git 的发布流程中。内容提交后触发重新构建通知，新标签出现时发布变更日志，或让 n8n 为每个拉取请求创建审阅任务。
- **Docmost** 适合作为文档目标。新客户注册时创建入门清单，从监控事件发布事故记录，或根据工作流输出持续更新内部运行手册。
- **Jupyter Notebook** 可以支持数据和报表工作流。运行计划 notebook，在数据库更新后导出报表，或在长时间运行的分析完成时通知频道。
- **Serposcope** 可以为 SEO 工作流提供数据。发送排名变化提醒，创建每周搜索位置摘要，或在重要关键词下滑时创建内容任务。
- **MySQL/MariaDB**、**Nginx & PHP** 和 **LAMP Stack** 对内部工具很有用。使用 n8n 写入行、调用私有管理端点、运行计划维护检查，或在自定义应用收到新提交时通知团队。

用于文件、存储、同步和备份：

- **Nextcloud** 暴露 WebDAV 和广泛的应用生态。把上传的文件移动到审阅文件夹，在客户上传到达时通知团队，创建日历跟进事项，或把已批准的文档复制到项目空间。
- **SFTPGo** 包含 REST API、事件动作和登录钩子。当客户上传报告时触发文件处理工作流，把完成的导出移动到 Nextcloud，在合作伙伴传输到达时通知频道，或在截止日期后禁用临时用户。
- **File Browser Quantum**、**File Browser WebDAV** 和 **Kodbox** 暴露 REST API、WebDAV、命令钩子或存储插件等文件管理能力。监控共享文件夹，生成元数据，把完成的素材复制到归档存储，或在发布前路由文件进行审批。
- **Rclone** 可在对象存储、SFTP、WebDAV 和云端远程之间通过脚本操作。从 n8n 安排同步任务，在大规模迁移后验证传输，把处理完成的媒体移入长期存储，或把失败摘要发送到聊天。
- **Restic REST Server** 为 restic 客户端提供 HTTPS 备份目标。计划备份检查失败时通知管理员，快照缺失后创建事故，或把每日备份健康摘要写入 Docmost。
- **Syncthing** 和 **Resilio Sync** 适合文件事件流水线。文件夹完成同步时作出响应，把完成的投递复制到媒体库，或在预期文件未到达时提醒用户。
- **Pure-FTPd** 和其他 FTP/SFTP 风格端点可以加入旧式文件工作流。轮询新的合作伙伴文件，将其移动到 SFTPGo 或 Nextcloud，然后在处理完成后通知相应团队。

用于安全、访问和基础设施：

- **Vaultwarden** 暴露兼容 Bitwarden 的 API，并可选支持 SMTP 通知。为密码库审查创建提醒，在入职或离职任务到期时通知管理员，或协调跨团队的密码轮换清单。
- **OpenVPN** 和 **WireGuard VPN** 可以成为访问工作流的一部分。在凭据需要轮换时通知运维人员，为新的 VPN 用户创建清单任务，或通过 n8n 运行计划连接检查。
- **Ubuntu**、**Debian**、**Fedora**、**Rocky Linux**、**AlmaLinux**、**Arch Linux** 和 **Windows VPS** 应用提供可脚本化环境。使用 n8n 触发 SSH 命令、运行健康检查、收集报告文件，或在计划任务后发布维护摘要。
- **Ubuntu**、**Debian** 和 **Firefox remote desktop** 应用可以支持浏览器或桌面任务。为人工审阅工作安排提醒，捕获脚本输出，或把完成的桌面导出交接到文件存储。

用于监控、聊天和通知：

- **Uptime Kuma** 支持丰富的通知目标和 webhook。把宕机事件转换成 Mattermost 或 Rocket.Chat 告警，创建 OpenProject 事故，请 OpenClaw 生成简短事故摘要，或把每次恢复记录到状态电子表格中。
- **Mattermost** 和 **Rocket.Chat** 是有用的通知与审批端点。从任何工作流发送人类可读的更新，请团队批准请求，然后在有人响应后继续自动化。
- **RocketChat Hubot** 适合聊天命令。让队友从聊天中触发 n8n webhook，询问部署状态，或启动带审计记录的脚本化维护工作流。
- **Matterbridge** 连接 IRC、Matrix、XMPP、Slack、Discord、Microsoft Teams、Telegram 等。使用 n8n 只格式化一次运维事件，然后将其转发给需要看到它们的社区或房间。
- **ZNC** 和 **The Lounge** 可以成为 chat-ops 工作流的一部分。把重要 IRC 事件转发到现代团队频道，归档高亮内容，或在监控关键词出现时通知运维人员。

用于媒体和下载自动化：

- **Tautulli** 暴露 Plex 活动、通知脚本和 JSON API。发送新增媒体摘要，在串流出错时提醒，导出使用报告，或在存储超过限制时创建清理任务。
- **Plex**、**Jellyfin**、**Emby**、**Navidrome**、**Audiobookshelf**、**Calibre**、**Madsonic** 和 **SeAT** 可以为媒体库或社区工作流提供数据。宣布新媒体，在导入后刷新媒体库，创建收听或阅读摘要，将元数据同步到仪表板，或在跟踪的活动变化时通知社区。
- **Immich** 适合照片工作流。在共享相册变化时通知家人，把导出的相册复制到归档存储，或在大量移动端上传后触发备份检查。
- **AzuraCast** 包含 REST API 和 webhook。宣布曲目或直播节目变更，更新网站小组件，在计划播出前通知主持人，或在节目结束后归档播放列表元数据。
- **Icecast** 可以支持广播状态工作流。监控串流可用性，发布正在播放更新，或在源断开连接时提醒频道。
- **Flood**、**ruTorrent**、**qBittorrent**、**Transmission** 和 **Deluge** 暴露 RPC 或 Web API。监控已完成下载，把完成的文件移动到正确的媒体库，触发媒体扫描，或在请求的项目就绪时通知用户。
- **PufferPanel**、**TeamSpeak 3** 以及游戏/聊天服务器也能融入运维工作流。服务器需要关注时通知管理员，创建维护提醒，或在计划变更后发布面向玩家的公告。

你也可以通过内置节点、webhook、HTTP 请求、数据库连接和代码节点，将 n8n 与外部服务一起使用。

## 安装 n8n

打开 [n8n App Store 页面](/appstore/app/264)，选择你的 Appbox，然后开始安装。

在设置过程中，Appbox 会要求你为 n8n 创建第一个所有者账号。输入你想用于该账号的管理员电子邮件地址、密码、名字和姓氏。

应用准备就绪后，打开已安装应用页面上显示的 Login URL，并使用刚创建的所有者账号登录。

## 创建你的第一个工作流

进入 n8n 后，前往 **Overview** 并选择 **Create Workflow**。编辑器会在画布上打开，自动化中的每一步都由一个节点表示。

从一个简单的手动工作流开始：

1. 点击 **Add first step**。
2. 搜索 **Manual Trigger** 并添加它。
3. 点击触发器节点上的加号按钮。
4. 添加第二个节点，例如 **Edit Fields**、**HTTP Request** 或某个应用专用节点。
5. 在侧边面板中配置该节点。

n8n 的主工作区是画布。你会在这里添加节点、连接它们、整理工作流并运行测试执行。

## 测试工作流

点击 **Execute Workflow** 手动运行工作流。

运行完成后，点击每个节点查看它接收和返回的数据。在连接真实应用、webhook 或计划触发器之前，这是了解数据如何在工作流中流动的最快方式。

## 连接应用和凭据

对于应用集成，添加所需节点，并在 n8n 要求时创建凭据。

对于 Appbox 托管的应用，常见模式是：

1. 在你想自动化的应用中创建 API 令牌或 webhook。
2. 添加匹配的 n8n 节点；如果没有专用节点，则使用 **HTTP Request** 节点。
3. 将令牌存储在 n8n 凭据中。
4. 在添加分支、重试或更多步骤之前，先用一个小操作进行测试。

对于传入事件，使用 **Webhook** 触发器。n8n 会提供一个生产 webhook URL，外部服务可以通过 HTTPS 调用它。

## 一个简单的首个自动化

把这个作为你的第一个真实工作流来试试：

1. 添加一个 **Webhook** 触发器。
2. 添加一个 **HTTP Request** 节点来调用另一个服务或 Appbox 应用 API。
3. 添加一个 **IF** 节点，根据响应进行分支。
4. 向 Mattermost、Rocket.Chat 或另一个通知频道发送消息。

这为许多 n8n 自动化提供了核心模式：接收事件，获取或转换数据，做出决策，然后通知某个人或更新另一个系统。

## 关于规模的一点说明

Appbox 套餐被设计为紧凑的单容器 n8n 部署，并带有持久存储。它是个人、团队和内部自动化的绝佳起点。

如果你计划运行非常高的执行量、大型载荷，或业务关键型自动化中心，请仔细测试工作负载，并考虑从已安装应用页面提升应用资源。

## 开始构建

如果你有 Appbox，n8n 是让其他工具快速产生连接感的方式之一。

[在 Appbox 上部署 n8n](/appstore/app/264)，创建一个工作流，先手动运行一次，然后把它变成你希望应用本来就具备的自动化。
