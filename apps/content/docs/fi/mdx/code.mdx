`opendocs` käyttää [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code) -kirjastoa
koodien käsittelyyn MDX-tiedostoissa. Tämän avulla voit mukauttaa koodien näyttötapaa merkittävästi!

Lisätietoja saat virallisesta dokumentaatiosta ja esimerkeistä!

## Otsikot

**Esimerkki:**

```js title="apps/web/src/index.tsx"
import { App } from './App';
```

**Koodi:**

````mdx
```js title="apps/web/src/index.tsx"
import { App } from './App';
```
````

**Esimerkki:**

**Koodi:**

```mdx

```

## Kontekstitietoinen rivinsisäisen koodin korostus

Jos sinulla olisi esimerkiksi seuraava koodilohko:

```js
function getStringLength(str) {
  return str.length;
}
```

Kun viittaamme `getStringLength{:.entity.name.function}`-arvoon funktiona,
voimme värittää sen sen mukaisesti. Sama koskee `function{:.keyword}`-arvoa tai
`str{:.variable.parameter}` vs. `str{:.variable.other.object}` jne. Tämä mahdollistaa
rivinsisäisen koodin semanttisen linkittämisen lähimpään koodilohkoon, johon se viittaa.

**Koodi:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Sanaryhmän korostus tunnisteen mukaan

Lisää tunniste `#`-merkin jälkeen sanojen perään.
Tämän avulla voit värittää merkkejä eri tavalla annetun tunnisteen perusteella.

**Esimerkki:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Koodi:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

Tyylien muuttaminen tai uusien lisääminen:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
