**mscp** est une variante multi-thread de `scp` qui copie des fichiers simultanément via plusieurs connexions SSH (SFTP). Cela permet des transferts beaucoup plus rapides pour les gros fichiers et dossiers en utilisant des connexions parallèles.

## Prérequis

Avant d'utiliser mscp, vous devez installer SFTPgo sur votre serveur Appbox afin d'obtenir vos identifiants SFTP.

### Étape 1 : Installer SFTPgo

1. Rendez-vous sur la [page de l'app SFTPgo](https://www.appbox.co/appstore/app/232) dans l'App Store Appbox
2. Cliquez sur **Installer** pour ajouter SFTPgo à votre serveur
3. Une fois installé, vous recevrez vos informations de connexion :
   - **Nom d'utilisateur** : votre nom d'utilisateur SFTP
   - **Mot de passe** : votre mot de passe SFTP
   - **Nom d'hôte** : par exemple `sftpgo.yourserver.appboxes.co`
   - **Port SFTP** : par exemple `25557`
   - **Port FTP/FTPS** : par exemple `25555`
   - **URL WebDav** : pour l'accès WebDAV

## Installer mscp

Choisissez la méthode d'installation adaptée à votre système d'exploitation :

### macOS

#### Avec Homebrew (recommandé)

```bash
brew install upa/tap/mscp
```

#### Avec MacPorts

```bash
sudo port install mscp
```

### Ubuntu / Debian

```bash
sudo add-apt-repository ppa:upaa/mscp
sudo apt-get update
sudo apt-get install mscp
```

### RHEL / Fedora / CentOS / Rocky Linux / AlmaLinux

```bash
sudo dnf copr enable upaaa/mscp
sudo dnf install mscp
```

### Windows

mscp n'a pas de binaires Windows natifs, mais vous pouvez l'utiliser via WSL (Windows Subsystem for Linux) :

#### Étape 1 : Installer WSL

Ouvrez PowerShell en tant qu'administrateur et exécutez :

```powershell
wsl --install
```

Redémarrez votre ordinateur lorsque cela vous est demandé.

#### Étape 2 : Installer mscp dans WSL

Une fois WSL configuré (Ubuntu est la valeur par défaut), ouvrez le terminal WSL et exécutez :

```bash
sudo add-apt-repository ppa:upaa/mscp
sudo apt-get update
sudo apt-get install mscp
```

<Alert className="my-4" variant="warning">
  <AlertTitle className="font-bold">Chemins de fichiers Windows dans WSL</AlertTitle>
  <AlertDescription className="pl-2">
    Dans WSL, vos lecteurs Windows sont accessibles sous `/mnt/`. Par exemple, `C:\Users\YourName\Downloads` devient `/mnt/c/Users/YourName/Downloads`.
  </AlertDescription>
</Alert>

### Compiler depuis la source

Si aucun paquet n'est disponible pour votre système, vous pouvez compiler mscp depuis la source :

```bash
# Clone the repository
git clone https://github.com/upa/mscp.git
cd mscp

# Prepare patched libssh
git submodule update --init
patch -d libssh -p1 < patch/$(git -C libssh describe).patch

# Install build dependencies
bash ./scripts/install-build-deps.sh

# Build
mkdir build && cd build
cmake ..
make

# Install
sudo make install
```

## Utilisation de base

La syntaxe de mscp est similaire à celle de `scp`. Voici des usages courants.

<Alert className="my-4" variant="warning">
  <AlertTitle className="font-bold">Important : utilisez votre port SFTP</AlertTitle>
  <AlertDescription className="pl-2">
    Dans les exemples ci-dessous, `25557` représente le **port SFTP** de vos paramètres SFTPgo. Remplacez-le par le port SFTP réel affiché dans votre panneau Options SFTPgo.
  </AlertDescription>
</Alert>

### Uploader un fichier vers votre serveur

```bash
# -P specifies the SFTP port (use your SFTP port from SFTPgo settings)
mscp -P 25557 localfile.zip username@sftpgo.yourserver.appboxes.co:/path/to/destination/
```

### Uploader un dossier

Contrairement à `scp`, mscp n'exige pas l'option `-r` pour les dossiers :

```bash
# -P 25557 is the SFTP port from your SFTPgo Options
mscp -P 25557 /local/directory username@sftpgo.yourserver.appboxes.co:/remote/path/
```

### Télécharger un fichier depuis votre serveur

```bash
# Replace 25557 with your actual SFTP port
mscp -P 25557 username@sftpgo.yourserver.appboxes.co:/remote/file.zip /local/destination/
```

### Télécharger un dossier

```bash
mscp -P 25557 username@sftpgo.yourserver.appboxes.co:/remote/directory/ /local/path/
```

## Options avancées

### Utiliser plusieurs connexions

Par défaut, mscp utilise plusieurs connexions. Vous pouvez préciser le nombre de connexions avec `-n` :

```bash
mscp -P 25557 -n 8 largefile.zip username@sftpgo.yourserver.appboxes.co:/destination/
```

### Reprendre les transferts échoués (checkpointing)

mscp prend en charge le checkpointing pour reprendre les transferts interrompus :

```bash
# Enable checkpointing
mscp -P 25557 -C /path/to/checkpoint.json largefile.zip username@server:/destination/

# Resume a failed transfer
mscp -P 25557 -C /path/to/checkpoint.json -r
```

### Spécifier l'authentification par clé SSH

Si vous avez configuré l'authentification par clé SSH avec SFTPgo :

```bash
mscp -P 25557 -i ~/.ssh/your_key localfile.zip username@sftpgo.yourserver.appboxes.co:/destination/
```

### Limiter la bande passante

Pour limiter la bande passante de transfert (utile sur les connexions partagées) :

```bash
mscp -P 25557 -l 10M localfile.zip username@sftpgo.yourserver.appboxes.co:/destination/
```

## Exemple : flux complet

Voici un exemple complet utilisant les identifiants SFTPgo d'Appbox :

```bash
# Set your credentials (replace with your actual values from SFTPgo Options)
HOST="sftpgo.yourserver.appboxes.co"  # Your Hostname
PORT="25557"                           # Your SFTP Port (not FTP port!)
USER="your_username"                   # Your Username

# Upload a large file using 4 connections
mscp -P $PORT -n 4 ~/Downloads/large-backup.tar.gz $USER@$HOST:/backups/

# Upload an entire directory
mscp -P $PORT ~/Projects/my-website/ $USER@$HOST:/www/

# Download files from server
mscp -P $PORT $USER@$HOST:/media/movies/ ~/Downloads/movies/
```

## Conseils de performance

1. **Utilisez plus de connexions pour les gros fichiers** : augmentez la valeur de `-n` pour les gros transferts de fichiers (par exemple `-n 8` ou `-n 16`)
2. **Utilisez moins de connexions pour beaucoup de petits fichiers** : pour les dossiers contenant de nombreux petits fichiers, moins de connexions peut être plus efficace
3. **Activez la compression pour les fichiers texte** : mscp hérite de la compression SSH ; utilisez `-C` pour les transferts contenant beaucoup de texte
4. **Utilisez le checkpointing pour les très gros transferts** : activez toujours le checkpointing pour les transferts de plusieurs gigaoctets afin de pouvoir reprendre en cas d'interruption

## Dépannage

### Connexion refusée

Assurez-vous d'utiliser le bon port SFTP depuis vos paramètres SFTPgo (pas le port FTP) :

```bash
mscp -P 25557 ...  # Use your actual SFTP port
```

### Permission refusée

Vérifiez que votre nom d'utilisateur et votre mot de passe sont corrects. Vous pouvez d'abord tester avec un client SFTP standard :

```bash
sftp -P 25557 username@sftpgo.yourserver.appboxes.co
```

### Transferts lents

Essayez d'ajuster le nombre de connexions parallèles :

```bash
mscp -P 25557 -n 16 ...  # Increase connections
```
