`opendocs` MDX files में code को handle करने के लिए [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code) library का उपयोग करता है।
इससे आप code के display होने के तरीके को बहुत अधिक customize कर सकते हैं!

अधिक जानकारी के लिए official documentation और examples देखें!

## Context-aware inline code highlighting

उदाहरण के लिए, अगर आपके पास नीचे दिया गया code block हो:

```js
function getStringLength(str) {
  return str.length;
}
```

जब हम `getStringLength{:.entity.name.function}` को function के रूप में refer करते हैं,
तो हम उसे उसी तरह color कर सकते हैं। यही बात `function{:.keyword}`, या
`str{:.variable.parameter}` बनाम `str{:.variable.other.object}` आदि पर भी लागू होती है। इससे inline code को उस nearest code block से semantically link किया जा सकता है जिससे वह संबंधित है।

**Code:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Word group highlighting by id

शब्दों के बाद `#` और फिर id जोड़ें।
इससे दिए गए id के आधार पर characters को अलग-अलग रंगों में दिखाया जा सकता है।

**Example:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Code:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

styles बदलने या नए styles जोड़ने के लिए:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
