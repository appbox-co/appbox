`opendocs` usa la libreria [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code)
per gestire il codice nei file MDX. Questo ti permette di personalizzare notevolmente il modo in cui il codice viene visualizzato.

Per maggiori informazioni, consulta la documentazione ufficiale e gli esempi.

## Titoli

**Esempio:**

```js title="apps/web/src/index.tsx"
import { App } from './App';
```

**Codice:**

````mdx
```js title="apps/web/src/index.tsx"
import { App } from './App';
```
````

## Evidenziazione del codice inline sensibile al contesto

Ad esempio, se avessi il seguente blocco di codice:

```js
function getStringLength(str) {
  return str.length;
}
```

Quando ci riferiamo a `getStringLength{:.entity.name.function}` come funzione,
possiamo colorarla come tale. Lo stesso vale per `function{:.keyword}`, oppure
`str{:.variable.parameter}` rispetto a `str{:.variable.other.object}`, ecc. Questo consente
di collegare semanticamente il codice inline al blocco di codice più vicino a cui si riferisce.

**Codice:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Evidenziazione di gruppi di parole per id

Inserisci un id dopo `#` dopo le parole.
Questo ti permette di colorare i caratteri in modo diverso in base all'id fornito.

**Esempio:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Codice:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

Per modificare o aggiungere nuovi stili:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
