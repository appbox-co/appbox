`opendocs` は MDX ファイル内のコードを扱うために [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code) ライブラリを使用しています。
これにより、コードの表示方法を大きくカスタマイズできます。

詳しくは公式ドキュメントとサンプルをご覧ください。

## タイトル

**例:**

```js title="apps/web/src/index.tsx"
import { App } from './App';
```

**コード:**

````mdx
```js title="apps/web/src/index.tsx"
import { App } from './App';
```
````

## コンテキスト対応インラインコードハイライト

たとえば、次のコードブロックがあるとします。

```js
function getStringLength(str) {
  return str.length;
}
```

`getStringLength{:.entity.name.function}` を関数として参照すると、
そのように色付けできます。`function{:.keyword}` や、
`str{:.variable.parameter}` と `str{:.variable.other.object}` なども同様です。これにより、
インラインコードを参照先の最も近いコードブロックへ意味的に関連付けられます。

**コード:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## ID による単語グループハイライト

単語の後、`#` の後ろに ID を付けます。
これにより、指定した ID に基づいて文字を異なる色にできます。

**例:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**コード:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

スタイルを変更または追加するには、次を編集します。

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
