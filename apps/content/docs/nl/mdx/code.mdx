`opendocs` gebruikt de library [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code)
om code in MDX-bestanden te verwerken. Hierdoor kun je de manier waarop code wordt weergegeven uitgebreid aanpassen!

Zie de officiele documentatie en voorbeelden voor meer informatie!

## Contextbewuste inline code highlighting

Als je bijvoorbeeld het volgende codeblok had:

```js
function getStringLength(str) {
  return str.length;
}
```

Wanneer we naar `getStringLength{:.entity.name.function}` verwijzen als functie,
kunnen we die ook zo kleuren. Hetzelfde geldt voor `function{:.keyword}`, of
`str{:.variable.parameter}` versus `str{:.variable.other.object}`, enz. Hierdoor kan
inline code semantisch worden gekoppeld aan het dichtstbijzijnde codeblok waarnaar die verwijst.

**Code:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Woordgroep-highlighting op id

Plaats na de woorden een id na `#`.
Hiermee kun je tekens anders kleuren op basis van de opgegeven id.

**Voorbeeld:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Code:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

Om stijlen te wijzigen of nieuwe stijlen toe te voegen:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
