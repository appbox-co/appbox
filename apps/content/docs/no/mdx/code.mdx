`opendocs` bruker biblioteket [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code)
til å håndtere kode i MDX-filer. Dette lar deg tilpasse måten kode vises på i stor grad!

For mer informasjon kan du se den offisielle dokumentasjonen og eksemplene!

```mdx

```

## Kontekstbevisst utheving av inline-kode

For eksempel, hvis du hadde følgende kodeblokk:

```js
function getStringLength(str) {
  return str.length;
}
```

Når vi omtaler `getStringLength{:.entity.name.function}` som en funksjon,
kan vi fargelegge den som det. Det samme gjelder `function{:.keyword}`, eller
`str{:.variable.parameter}` kontra `str{:.variable.other.object}`, osv. Dette gjør det mulig
å knytte inline-kode semantisk til den nærmeste kodeblokken den refererer til.

**Kode:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Ordgruppeutheving etter id

Legg til en id etter `#` etter ordene.
Dette lar deg fargelegge tegn ulikt basert på den angitte id-en.

**Eksempel:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Kode:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

For å endre eller legge til nye stiler:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
