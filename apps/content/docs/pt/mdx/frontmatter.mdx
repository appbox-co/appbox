O frontmatter é uma parte importante de um arquivo MDX.
Ele é usado para definir informações sobre o conteúdo do arquivo,
como título, descrição, autor, data de publicação etc.

O cabeçalho de metadados é definido no início do arquivo, entre `---` (três hífens) no começo e no fim.

## Formato

O cabeçalho de metadados é definido no formato chave-valor, em que a chave é o nome do metadado e o valor é o conteúdo desse metadado.

**Exemplo:**

```mdx
---
title: Document title
description: Document description
---
```

Dependendo do tipo de documento (documentação, post de blog etc.), diferentes metadados podem ser usados.

## Metadados para documentação

- `title`
- `description`
- `links`
  - `source`
  - `docs`
  - `blog`
  - `api`

**Exemplo:**

```mdx
---
title: Document title
description: Document description
links:
  source: https://google.com
  docs: /docs
  blog: /blog
  api: https://google.com
---
```

## Metadados para posts de blog

<Alert className="my-4" variant="warning">
  <AlertTitle className="font-bold">`author_id`</AlertTitle>

  <AlertDescription className="pl-2">
    É o identificador do autor do post.
    Ele deve corresponder ao `id` do autor definido no arquivo `src/config/blog.ts`,
    que é o arquivo que contém os dados completos dos autores.
  </AlertDescription>
</Alert>

<Alert className="my-4" variant="warning">
  <AlertTitle className="font-bold">`og_image`</AlertTitle>

  <AlertDescription className="pl-2">
    Esta é a imagem que será usada como imagem de destaque do post quando ele for compartilhado nas redes sociais, caso você não queira usar a imagem gerada automaticamente.
    As imagens devem ser armazenadas na pasta `public/blog-og`.
    Use apenas o nome do arquivo de imagem e sua extensão.
  </AlertDescription>
</Alert>

- `title`
- `excerpt`
- `date`
- `author_id`
- `og_image`
- `tags`
- `links`
  - `source`
  - `docs`
  - `blog`
  - `api`

**Exemplo:**

```mdx
---
title: Post title
date: 2024-08-07 22:33:00
excerpt: Post excerpt
author_id: daltonmenezes
og_image: introducing-blogs-og.jpg
links:
  source: https://google.com
  docs: /docs
  blog: /blog
  api: https://google.com
tags: [next.js, open-graph, blog]
---
```
