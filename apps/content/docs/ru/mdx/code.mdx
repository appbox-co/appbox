`opendocs` использует библиотеку [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code)
для обработки кода в MDX-файлах. Это позволяет гибко настраивать отображение кода!

Подробнее смотрите в официальной документации и примерах.

## Заголовки

**Пример:**

```js title="apps/web/src/index.tsx"
import { App } from './App';
```

**Код:**

````mdx
```js title="apps/web/src/index.tsx"
import { App } from './App';
```
````

## Контекстная подсветка inline code

Например, если у вас есть следующий блок кода:

```js
function getStringLength(str) {
  return str.length;
}
```

Когда мы ссылаемся на `getStringLength{:.entity.name.function}` как на функцию,
мы можем подсветить ее соответствующим образом. То же касается `function{:.keyword}` или
`str{:.variable.parameter}` в сравнении с `str{:.variable.other.object}` и т. д. Это позволяет
семантически связывать inline code с ближайшим блоком кода, к которому он относится.

**Код:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Групповая подсветка слов по id

Укажите id после `#` после слов.
Это позволяет раскрашивать символы по-разному в зависимости от заданного id.

**Пример:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Код:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

Чтобы изменить или добавить новые стили:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
