## OznaÄŤevanje vrstiÄŤne kode z upoĹˇtevanjem konteksta

Na primer, ÄŤe bi imeli naslednji blok kode:

```js

  return str.length;
}
```

Ko `getStringLength{:.entity.name.function}` omenjamo kot funkcijo,
ga lahko tako tudi obarvamo. Enako velja za `function{:.keyword}` ali
`str{:.variable.parameter}` v primerjavi z `str{:.variable.other.object}` itd. To omogoÄŤa
semantiÄŤno povezovanje vrstiÄŤne kode z najbliĹľjim blokom kode, na katerega se nanaĹˇa.

**Koda:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## OznaÄŤevanje skupin besed po id

Za besedami dodajte id za `#`.
To omogoÄŤa razliÄŤno barvanje znakov glede na podani id.

**Primer:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i

const [name, setName] = useState("Taylor");
```

**Koda:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

```css title="apps/web/src/styles/mdx.css"

  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
