`opendocs` använder biblioteket [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code)
för att hantera kod i MDX-filer. Det gör att du kan anpassa hur kod visas i stor utsträckning!

Mer information finns i den officiella dokumentationen och exemplen!

## Titlar

**Exempel:**

```js title="apps/web/src/index.tsx"
import { App } from './App';
```

**Kod:**

````mdx
```js title="apps/web/src/index.tsx"
import { App } from './App';
```
````


## Kontextmedveten markering av inline-kod

Om du till exempel hade följande kodblock:

```js
function getStringLength(str) {
  return str.length;
}
```

När vi hänvisar till `getStringLength{:.entity.name.function}` som en funktion
kan vi färglägga den som sådan. Detsamma gäller `function{:.keyword}`, eller
`str{:.variable.parameter}` kontra `str{:.variable.other.object}` osv. Det gör att
inline-kod semantiskt kan kopplas till det närmaste kodblock den hänvisar till.

**Kod:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Ordgruppsmarkering efter id

Placera ett id efter `#` efter orden.
Det gör att du kan färglägga tecken olika baserat på angivet id.

**Exempel:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Kod:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

För att ändra eller lägga till nya stilar:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
