MDX ఫైల్‌లలో code ను నిర్వహించడానికి `opendocs` [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code) library ను ఉపయోగిస్తుంది.
దీంతో code ఎలా కనిపించాలో మీరు చాలా వరకు customize చేయగలరు!

మరింత సమాచారం కోసం, అధికారిక documentation మరియు examples చూడండి!

```
````

## Context-aware inline code highlighting

ఉదాహరణకు, మీ వద్ద క్రింది code block ఉంటే:

```js
function getStringLength(str) {
  return str.length;
}
```

`getStringLength{:.entity.name.function}` ను function గా సూచించినప్పుడు,
దానిని ఆ విధంగా color చేయవచ్చు. `function{:.keyword}` విషయంలోనూ అలాగే,
లేదా `str{:.variable.parameter}` vs. `str{:.variable.other.object}` మొదలైనవి. ఇది inline code ను అది సూచించే సమీప code block తో semantically link చేయడానికి అనుమతిస్తుంది.

**Code:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## ID ద్వారా word group highlighting

పదాల తర్వాత `#` తర్వాత id పెట్టండి.
ఇది ఇచ్చిన id ఆధారంగా characters ను వేర్వేరు రంగులతో చూపడానికి అనుమతిస్తుంది.

**ఉదాహరణ:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Code:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

కొత్త styles మార్చడానికి లేదా జోడించడానికి:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
