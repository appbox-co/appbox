`opendocs`, MDX dosyalarındaki kodları işlemek için [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code) kütüphanesini kullanır.
Bu, kodların nasıl görüntüleneceğini büyük ölçüde özelleştirmenizi sağlar!

Daha fazla bilgi için resmi dokümantasyona ve örneklere bakın!

## Başlıklar

**Örnek:**

```js title="apps/web/src/index.tsx"
import { App } from './App';
```

**Kod:**

````mdx
```js title="apps/web/src/index.tsx"
import { App } from './App';
```
````

## Satır içi kod vurgulama

**Örnek:**

`[1, 2, 3].join('-'){:js}` sonucu `'1-2-3'{:js}` olur.

**Kod:**

```mdx
The result of `[1, 2, 3].join('-'){:js}` is `'1-2-3'{:js}`.
```

## Bağlama duyarlı satır içi kod vurgulama

Örneğin, aşağıdaki kod bloğunuz varsa:

```js
function getStringLength(str) {
  return str.length;
}
```

`getStringLength{:.entity.name.function}` değerinden bir fonksiyon olarak söz ettiğimizde,
onu buna göre renklendirebiliriz. Aynısı `function{:.keyword}` veya
`str{:.variable.parameter}` ile `str{:.variable.other.object}` vb. için de geçerlidir. Bu, satır içi kodu başvurduğu en yakın kod bloğuna anlamsal olarak bağlamayı sağlar.

**Kod:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Id'ye göre sözcük grubu vurgulama

Sözcüklerden sonra `#` işaretinin ardından bir id koyun.
Bu, karakterleri verilen id'ye göre farklı şekilde renklendirmenizi sağlar.

**Örnek:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Kod:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

Stilleri değiştirmek veya yenilerini eklemek için:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
