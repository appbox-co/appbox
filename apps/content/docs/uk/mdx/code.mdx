`opendocs` використовує бібліотеку [`rehype-pretty-code`](https://github.com/atomiks/rehype-pretty-code)
для обробки коду в MDX файлах. Це дає змогу значно налаштовувати спосіб відображення коду!

Щоб дізнатися більше, перегляньте офіційну документацію та приклади!

## Контекстне підсвічування inline code

Наприклад, якщо у вас є такий блок коду:

```js
function getStringLength(str) {
  return str.length;
}
```

Коли ми посилаємося на `getStringLength{:.entity.name.function}` як на функцію,
ми можемо забарвити його відповідно. Те саме стосується `function{:.keyword}` або
`str{:.variable.parameter}` проти `str{:.variable.other.object}` тощо. Це дає змогу
семантично пов’язувати inline code з найближчим блоком коду, до якого він належить.

**Код:**
````mdx
When we refer to `getStringLength{:.entity.name.function}` as a function,
we can color it as such. Same with `function{:.keyword}`, or
`str{:.variable.parameter}` vs. `str{:.variable.other.object}`, etc. This allows
semantically link inline code to the nearest block of code it refers to.
````

## Підсвічування груп слів за id

Додайте id після `#` після слів.
Це дає змогу забарвлювати символи по-різному залежно від заданого id.

**Приклад:**

```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```

**Код:**
````md
```js /age/#v /name/#v /setAge/#s /setName/#s /50/#i /"Taylor"/#i
const [age, setAge] = useState(50);
const [name, setName] = useState("Taylor");
```
````

Щоб змінити або додати нові стилі:

```css title="apps/web/src/styles/mdx.css"
[data-chars-id='v'] {
  @apply !text-pink-300 bg-rose-800/50 font-bold;
}
```
