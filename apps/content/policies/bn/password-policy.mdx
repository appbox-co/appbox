Appbox-এ আমরা পাসওয়ার্ড নিরাপত্তাকে অত্যন্ত গুরুত্ব দিই এবং আপনার লগইন তথ্য সুরক্ষার জন্য একাধিক স্তরের সুরক্ষা প্রয়োগ করি। এই নীতিতে আমরা কীভাবে পাসওয়ার্ড পরিচালনা করি তা ব্যাখ্যা করা হয়েছে এবং আপনার Appbox পরিষেবাগুলোর জন্য শক্তিশালী ও নিরাপদ পাসওয়ার্ড তৈরি করতে সহায়ক নির্দেশনা দেওয়া হয়েছে।

## Appbox-এ পাসওয়ার্ড সংরক্ষণ

Appbox ব্যবহারকারীর অভিজ্ঞতা সহজ ও নিরবচ্ছিন্ন রাখতে শিল্প-মানের নিরাপত্তা প্ল্যাটফর্ম ব্যবহার করে। নিচের অংশগুলোতে আমাদের বিভিন্ন সিস্টেমে কীভাবে পাসওয়ার্ড পরিচালনা করা হয় তা বিস্তারিতভাবে ব্যাখ্যা করা হয়েছে।

### ক্লায়েন্ট এরিয়া (WHMCS)

WHMCS দ্বারা পরিচালিত Appbox Client Area অ্যাকাউন্ট ব্যবস্থাপনা, পরিষেবা প্রভিশনিং, যোগাযোগ এবং পেমেন্ট পরিচালনা করে। এই সিস্টেমে আপনার পাসওয়ার্ড Bcrypt ব্যবহার করে নিরাপদে হ্যাশ করা হয়, যা পাসওয়ার্ড নিরাপত্তার জন্য বিশেষভাবে তৈরি একটি আধুনিক ক্রিপ্টোগ্রাফিক অ্যালগরিদম।

### Appbox কন্ট্রোল প্যানেল

আমাদের Control Panel আপনার Appbox পরিষেবা, ইনস্টল করা অ্যাপ্লিকেশন এবং অ্যাকাউন্ট সেটিংস পরিচালনার জন্য একটি কেন্দ্রীভূত ইন্টারফেস দেয়। এটি আধুনিক অথেন্টিকেশন মানসহ নিরাপদ ফ্রেমওয়ার্ক ব্যবহার করে তৈরি।

### Control Panel লগইন পাসওয়ার্ড

Appbox Control Panel-এ প্রবেশ করতে আপনি যে পাসওয়ার্ড ব্যবহার করেন, তা Blowfish symmetric block cipher-ভিত্তিক একটি adaptive hash function bcrypt দিয়ে সুরক্ষিত। আমাদের বাস্তবায়নে Phalcon security component ব্যবহার করা হয়েছে, যা সর্বোত্তম নিরাপত্তা নিশ্চিত করতে শক্তিশালী work factor সহ PHP-এর `password_hash` ফাংশন ব্যবহার করে। এর অর্থ:

1. Rainbow table আক্রমণ প্রতিরোধ করতে প্রতিটি পাসওয়ার্ড হ্যাশে একটি অনন্য salt থাকে
2. Brute force প্রচেষ্টা কমাতে হ্যাশিং অ্যালগরিদম ইচ্ছাকৃতভাবে ধীর
3. আক্রমণকে গণনাগতভাবে ব্যয়বহুল করতে work factor (cost) নির্ধারণ করা হয়
4. উন্নত নিরাপত্তার প্রয়োজন হলে আমাদের সিস্টেম Argon2i-এর মতো নতুন অ্যালগরিদমে আপগ্রেড করতে পারে

ডাটাবেস আপসের ঘটনাতেও এই শিল্প-মানের পদ্ধতি পাসওয়ার্ড প্রকাশ হওয়া প্রতিরোধ করে।

### অ্যাপ্লিকেশন পাসওয়ার্ড

আমাদের Control Panel-এর মাধ্যমে ইনস্টল করা অ্যাপ্লিকেশনগুলোর পাসওয়ার্ড plaintext আকারে সংরক্ষিত থাকে এবং আপনি ও Appbox সাপোর্ট টিম উভয়ের জন্যই অ্যাক্সেসযোগ্য। আপনি সহায়তা চাইলে প্রযুক্তিগত সহায়তা দেওয়ার জন্য এই অ্যাক্সেসযোগ্যতা প্রয়োজন। আমাদের সাপোর্ট টিম শুধুমাত্র তখনই আপনার অ্যাপ্লিকেশনে অ্যাক্সেস করবে, যখন আপনি আমাদের অফিসিয়াল সাপোর্ট চ্যানেলের মাধ্যমে স্পষ্টভাবে সাহায্য চাইবেন। কোনো নির্দিষ্ট অ্যাপ্লিকেশনের জন্য সাপোর্ট প্রয়োজন হলে আমরা অস্থায়ী পাসওয়ার্ড সেট করার পরামর্শ দিই।

### তৃতীয়-পক্ষ অ্যাপ্লিকেশন পাসওয়ার্ড

কিছু অ্যাপ্লিকেশন তাদের ডিফল্ট কার্যকারিতার অংশ হিসেবে অথেন্টিকেশন ক্রেডেনশিয়াল এবং সংবেদনশীল তথ্য plaintext আকারে সংরক্ষণ করতে পারে। এই আচরণ Appbox-এর নিয়ন্ত্রণের বাইরে এবং সংশ্লিষ্ট অ্যাপ্লিকেশনের ডেভেলপাররা এটি নির্ধারণ করেন। আপনার Appbox পরিবেশ এমনভাবে সুরক্ষিত যে শুধুমাত্র আপনিই আপনার নিজের ডেটা অ্যাক্সেস করতে পারেন, ফলে অ্যাপ্লিকেশন-স্তরের পাসওয়ার্ড সংরক্ষণ সীমাবদ্ধতা থাকলেও ব্যবহারকারীদের মধ্যে পৃথকীকরণ বজায় থাকে।

## পাসওয়ার্ড নিরাপত্তা সুপারিশ

নিচের পাসওয়ার্ড-সংক্রান্ত সর্বোত্তম চর্চাগুলো অনুসরণ করে আপনি আপনার অ্যাকাউন্টের নিরাপত্তা আরও শক্তিশালী করতে পারেন:

### অনন্য পাসওয়ার্ড ব্যবহার করুন

* বিভিন্ন ওয়েবসাইট বা পরিষেবায় কখনও একই পাসওয়ার্ড পুনরায় ব্যবহার করবেন না
* একাধিক অ্যাকাউন্টে একই পাসওয়ার্ড ব্যবহার করলে বড় নিরাপত্তা ঝুঁকি তৈরি হয় - একটি পরিষেবা আপস হলে আপনার সব অ্যাকাউন্ট ঝুঁকিতে পড়ে
* আপনার প্রতিটি Appbox পরিষেবার জন্য আলাদা পাসওয়ার্ড তৈরি করুন

### শক্তিশালী, মনে রাখার মতো পাসওয়ার্ড তৈরি করুন

* কমপক্ষে ১২ অক্ষরের পাসওয়ার্ড ব্যবহার করার চেষ্টা করুন
* ব্যবহার করার কথা বিবেচনা করতে পারেন:
  * কোনো সিনেমা বা বইয়ের মনে রাখার মতো উক্তি
  * একটি অনন্য বাক্যাংশ বা অভিব্যক্তি
  * বিশেষ অক্ষরসহ সম্পর্কহীন শব্দের সমন্বয়
* ব্যবহার এড়িয়ে চলুন:
  * ব্যক্তিগত তথ্য (জন্মদিন, নাম, আদ্যক্ষর)
  * প্রচলিত শব্দ বা বাক্যাংশ
  * সহজ কিবোর্ড প্যাটার্ন (qwerty, 12345)
  * উল্টো করা শব্দ বা সহজ প্রতিস্থাপন

### দুই-ধাপের অথেন্টিকেশন চালু করুন

আমরা আপনার Appbox অ্যাকাউন্টে two-factor authentication (2FA) চালু করার জোরালো পরামর্শ দিই:

* 2FA আপনার পাসওয়ার্ডের বাইরে একটি অত্যাবশ্যক দ্বিতীয় নিরাপত্তা স্তর যোগ করে
* আপনার পাসওয়ার্ড আপস হলেও, আক্রমণকারীদের আপনার অ্যাকাউন্টে প্রবেশ করতে দ্বিতীয় factor প্রয়োজন হবে
* আপনি আপনার [account security page](https://billing.appbox.co/user/security)-এ 2FA চালু করতে পারেন
* Appbox নিচের অথেন্টিকেশন অ্যাপসহ time-based token সমর্থন করে:
  * Google Authenticator
  * Authy
  * Microsoft Authenticator 
  * Duo Mobile
  * এবং অধিকাংশ অন্যান্য মানসম্মত TOTP-compatible অ্যাপ্লিকেশন

### Password Manager ব্যবহার করার কথা বিবেচনা করুন

আপনার সব অ্যাকাউন্টের জন্য অনন্য পাসওয়ার্ড তৈরি ও মনে রাখতে অসুবিধা হলে একটি নির্ভরযোগ্য password manager ব্যবহার করার কথা বিবেচনা করুন। এসব টুল আপনার জন্য শক্তিশালী পাসওয়ার্ড তৈরি করতে এবং নিরাপদে সংরক্ষণ করতে পারে। কিছু নির্ভরযোগ্য বিকল্প হলো:

* [Bitwarden](https://bitwarden.com/)
* [LastPass](https://www.lastpass.com/)
* [LessPass](https://lesspass.com/)
* [KeePass](https://keepass.info/)

## দুর্বল পাসওয়ার্ড নীতি বিবৃতি

Appbox শক্তিশালী পাসওয়ার্ড ব্যবহারে উৎসাহিত করলেও, শেষ পর্যন্ত আপনার অ্যাকাউন্ট ক্রেডেনশিয়ালের নিরাপত্তার দায়িত্ব আপনার। **দুর্বল পাসওয়ার্ড বা একাধিক সাইটে পাসওয়ার্ড পুনরায় ব্যবহার করার কারণে আপনার অ্যাকাউন্ট আপস হলে Appbox-কে দায়ী করা যাবে না।** আক্রমণকারীরা দুর্বল পাসওয়ার্ডযুক্ত অ্যাকাউন্ট আপস করতে সাধারণত brute force বা dictionary attack ব্যবহার করে। আপনার অ্যাকাউন্ট ও ডেটা সুরক্ষিত রাখতে ওপরের নিরাপত্তা নির্দেশনাগুলো অনুসরণ করার জন্য আমরা জোরালো পরামর্শ দিই।

Appbox-এ আমরা পাসওয়ার্ড নিরাপত্তাকে অত্যন্ত গুরুত্ব দিই এবং আপনার লগইন তথ্য সুরক্ষার জন্য একাধিক স্তরের সুরক্ষা প্রয়োগ করি। এই নীতিতে আমরা কীভাবে পাসওয়ার্ড পরিচালনা করি তা ব্যাখ্যা করা হয়েছে এবং আপনার Appbox পরিষেবাগুলোর জন্য শক্তিশালী ও নিরাপদ পাসওয়ার্ড তৈরি করতে সহায়ক নির্দেশনা দেওয়া হয়েছে।

## Appbox-এ পাসওয়ার্ড সংরক্ষণ

Appbox ব্যবহারকারীর অভিজ্ঞতা সহজ ও নিরবচ্ছিন্ন রাখতে শিল্প-মানের নিরাপত্তা প্ল্যাটফর্ম ব্যবহার করে। নিচের অংশগুলোতে আমাদের বিভিন্ন সিস্টেমে কীভাবে পাসওয়ার্ড পরিচালনা করা হয় তা বিস্তারিতভাবে ব্যাখ্যা করা হয়েছে।

### ক্লায়েন্ট এরিয়া (WHMCS)

WHMCS দ্বারা পরিচালিত Appbox Client Area অ্যাকাউন্ট ব্যবস্থাপনা, পরিষেবা প্রভিশনিং, যোগাযোগ এবং পেমেন্ট পরিচালনা করে। এই সিস্টেমে আপনার পাসওয়ার্ড Bcrypt ব্যবহার করে নিরাপদে হ্যাশ করা হয়, যা পাসওয়ার্ড নিরাপত্তার জন্য বিশেষভাবে তৈরি একটি আধুনিক ক্রিপ্টোগ্রাফিক অ্যালগরিদম।

### Appbox কন্ট্রোল প্যানেল

আমাদের Control Panel আপনার Appbox পরিষেবা, ইনস্টল করা অ্যাপ্লিকেশন এবং অ্যাকাউন্ট সেটিংস পরিচালনার জন্য একটি কেন্দ্রীভূত ইন্টারফেস দেয়। এটি আধুনিক অথেন্টিকেশন মানসহ নিরাপদ ফ্রেমওয়ার্ক ব্যবহার করে তৈরি।

### Control Panel লগইন পাসওয়ার্ড

Appbox Control Panel-এ প্রবেশ করতে আপনি যে পাসওয়ার্ড ব্যবহার করেন, তা Blowfish symmetric block cipher-ভিত্তিক একটি adaptive hash function bcrypt দিয়ে সুরক্ষিত। আমাদের বাস্তবায়নে Phalcon security component ব্যবহার করা হয়েছে, যা সর্বোত্তম নিরাপত্তা নিশ্চিত করতে শক্তিশালী work factor সহ PHP-এর `password_hash` ফাংশন ব্যবহার করে। এর অর্থ:

1. Rainbow table আক্রমণ প্রতিরোধ করতে প্রতিটি পাসওয়ার্ড হ্যাশে একটি অনন্য salt থাকে
2. Brute force প্রচেষ্টা কমাতে হ্যাশিং অ্যালগরিদম ইচ্ছাকৃতভাবে ধীর
3. আক্রমণকে গণনাগতভাবে ব্যয়বহুল করতে work factor (cost) নির্ধারণ করা হয়
4. উন্নত নিরাপত্তার প্রয়োজন হলে আমাদের সিস্টেম Argon2i-এর মতো নতুন অ্যালগরিদমে আপগ্রেড করতে পারে

ডাটাবেস আপসের ঘটনাতেও এই শিল্প-মানের পদ্ধতি পাসওয়ার্ড প্রকাশ হওয়া প্রতিরোধ করে।

### অ্যাপ্লিকেশন পাসওয়ার্ড

আমাদের Control Panel-এর মাধ্যমে ইনস্টল করা অ্যাপ্লিকেশনগুলোর পাসওয়ার্ড plaintext আকারে সংরক্ষিত থাকে এবং আপনি ও Appbox সাপোর্ট টিম উভয়ের জন্যই অ্যাক্সেসযোগ্য। আপনি সহায়তা চাইলে প্রযুক্তিগত সহায়তা দেওয়ার জন্য এই অ্যাক্সেসযোগ্যতা প্রয়োজন। আমাদের সাপোর্ট টিম শুধুমাত্র তখনই আপনার অ্যাপ্লিকেশনে অ্যাক্সেস করবে, যখন আপনি আমাদের অফিসিয়াল সাপোর্ট চ্যানেলের মাধ্যমে স্পষ্টভাবে সাহায্য চাইবেন। কোনো নির্দিষ্ট অ্যাপ্লিকেশনের জন্য সাপোর্ট প্রয়োজন হলে আমরা অস্থায়ী পাসওয়ার্ড সেট করার পরামর্শ দিই।

### তৃতীয়-পক্ষ অ্যাপ্লিকেশন পাসওয়ার্ড

কিছু অ্যাপ্লিকেশন তাদের ডিফল্ট কার্যকারিতার অংশ হিসেবে অথেন্টিকেশন ক্রেডেনশিয়াল এবং সংবেদনশীল তথ্য plaintext আকারে সংরক্ষণ করতে পারে। এই আচরণ Appbox-এর নিয়ন্ত্রণের বাইরে এবং সংশ্লিষ্ট অ্যাপ্লিকেশনের ডেভেলপাররা এটি নির্ধারণ করেন। আপনার Appbox পরিবেশ এমনভাবে সুরক্ষিত যে শুধুমাত্র আপনিই আপনার নিজের ডেটা অ্যাক্সেস করতে পারেন, ফলে অ্যাপ্লিকেশন-স্তরের পাসওয়ার্ড সংরক্ষণ সীমাবদ্ধতা থাকলেও ব্যবহারকারীদের মধ্যে পৃথকীকরণ বজায় থাকে।

## পাসওয়ার্ড নিরাপত্তা সুপারিশ

নিচের পাসওয়ার্ড-সংক্রান্ত সর্বোত্তম চর্চাগুলো অনুসরণ করে আপনি আপনার অ্যাকাউন্টের নিরাপত্তা আরও শক্তিশালী করতে পারেন:

### অনন্য পাসওয়ার্ড ব্যবহার করুন

* বিভিন্ন ওয়েবসাইট বা পরিষেবায় কখনও একই পাসওয়ার্ড পুনরায় ব্যবহার করবেন না
* একাধিক অ্যাকাউন্টে একই পাসওয়ার্ড ব্যবহার করলে বড় নিরাপত্তা ঝুঁকি তৈরি হয় - একটি পরিষেবা আপস হলে আপনার সব অ্যাকাউন্ট ঝুঁকিতে পড়ে
* আপনার প্রতিটি Appbox পরিষেবার জন্য আলাদা পাসওয়ার্ড তৈরি করুন

### শক্তিশালী, মনে রাখার মতো পাসওয়ার্ড তৈরি করুন

* কমপক্ষে ১২ অক্ষরের পাসওয়ার্ড ব্যবহার করার চেষ্টা করুন
* ব্যবহার করার কথা বিবেচনা করতে পারেন:
  * কোনো সিনেমা বা বইয়ের মনে রাখার মতো উক্তি
  * একটি অনন্য বাক্যাংশ বা অভিব্যক্তি
  * বিশেষ অক্ষরসহ সম্পর্কহীন শব্দের সমন্বয়
* ব্যবহার এড়িয়ে চলুন:
  * ব্যক্তিগত তথ্য (জন্মদিন, নাম, আদ্যক্ষর)
  * প্রচলিত শব্দ বা বাক্যাংশ
  * সহজ কিবোর্ড প্যাটার্ন (qwerty, 12345)
  * উল্টো করা শব্দ বা সহজ প্রতিস্থাপন

### দুই-ধাপের অথেন্টিকেশন চালু করুন

আমরা আপনার Appbox অ্যাকাউন্টে two-factor authentication (2FA) চালু করার জোরালো পরামর্শ দিই:

* 2FA আপনার পাসওয়ার্ডের বাইরে একটি অত্যাবশ্যক দ্বিতীয় নিরাপত্তা স্তর যোগ করে
* আপনার পাসওয়ার্ড আপস হলেও, আক্রমণকারীদের আপনার অ্যাকাউন্টে প্রবেশ করতে দ্বিতীয় factor প্রয়োজন হবে
* আপনি আপনার [account security page](https://billing.appbox.co/user/security)-এ 2FA চালু করতে পারেন
* Appbox নিচের অথেন্টিকেশন অ্যাপসহ time-based token সমর্থন করে:
  * Google Authenticator
  * Authy
  * Microsoft Authenticator 
  * Duo Mobile
  * এবং অধিকাংশ অন্যান্য মানসম্মত TOTP-compatible অ্যাপ্লিকেশন

### Password Manager ব্যবহার করার কথা বিবেচনা করুন

আপনার সব অ্যাকাউন্টের জন্য অনন্য পাসওয়ার্ড তৈরি ও মনে রাখতে অসুবিধা হলে একটি নির্ভরযোগ্য password manager ব্যবহার করার কথা বিবেচনা করুন। এসব টুল আপনার জন্য শক্তিশালী পাসওয়ার্ড তৈরি করতে এবং নিরাপদে সংরক্ষণ করতে পারে। কিছু নির্ভরযোগ্য বিকল্প হলো:

* [Bitwarden](https://bitwarden.com/)
* [LastPass](https://www.lastpass.com/)
* [LessPass](https://lesspass.com/)
* [KeePass](https://keepass.info/)

## দুর্বল পাসওয়ার্ড নীতি বিবৃতি

Appbox শক্তিশালী পাসওয়ার্ড ব্যবহারে উৎসাহিত করলেও, শেষ পর্যন্ত আপনার অ্যাকাউন্ট ক্রেডেনশিয়ালের নিরাপত্তার দায়িত্ব আপনার। **দুর্বল পাসওয়ার্ড বা একাধিক সাইটে পাসওয়ার্ড পুনরায় ব্যবহার করার কারণে আপনার অ্যাকাউন্ট আপস হলে Appbox-কে দায়ী করা যাবে না।** আক্রমণকারীরা দুর্বল পাসওয়ার্ডযুক্ত অ্যাকাউন্ট আপস করতে সাধারণত brute force বা dictionary attack ব্যবহার করে। আপনার অ্যাকাউন্ট ও ডেটা সুরক্ষিত রাখতে ওপরের নিরাপত্তা নির্দেশনাগুলো অনুসরণ করার জন্য আমরা জোরালো পরামর্শ দিই।

Appbox-এ আমরা password security গুরুত্বের সাথে নিই এবং আপনার credentials-এর জন্য multiple layers of protection implement করি। এই policy আমরা passwords কীভাবে handle করি তা explain করে এবং আপনার Appbox services-এর জন্য strong, secure passwords তৈরি করতে guidelines দেয়।

## Password Storage at Appbox

Seamless user experience দিতে Appbox industry-standard security platforms ব্যবহার করে। নিচের sections আমাদের different systems জুড়ে passwords কীভাবে manage করি তা detail করে।

### Client Area (WHMCS)

WHMCS-powered Appbox Client Area account management, service provisioning, communications, এবং payments handle করে। এই system-এ আপনার password Bcrypt ব্যবহার করে securely hashed, যা password security-এর জন্য specifically designed modern cryptographic algorithm।

### Appbox Control Panel

আমাদের Control Panel আপনার Appbox services, installed applications, এবং account settings manage করার centralized interface দেয়। এটি modern authentication standards সহ secure frameworks ব্যবহার করে built।

### Control Panel Login Password

Appbox Control Panel access করতে আপনি যে password ব্যবহার করেন তা bcrypt দিয়ে protected, যা Blowfish symmetric block cipher-এর ওপর ভিত্তি করা adaptive hash function। আমাদের implementation Phalcon security component leverage করে, যা optimal security নিশ্চিত করতে strong work factors সহ PHP-এর `password_hash` function ব্যবহার করে। এর অর্থ:

1. Rainbow table attacks prevent করতে প্রতিটি password hash-এ unique salt থাকে
2. Brute force attempts mitigate করতে hashing algorithm deliberately slow
3. Attacks computationally expensive করতে work factor (cost) set করা
4. Enhanced security-এর জন্য প্রয়োজন হলে আমাদের system Argon2i-এর মতো newer algorithms-এ upgrade করতে পারে

এই industry-standard approach database compromise হলেও password exposure prevent করে।

### Application Passwords

আমাদের Control Panel দিয়ে installed applications-এর passwords plaintext-এ stored এবং আপনার ও Appbox support team উভয়ের কাছে accessible। আপনি assistance request করলে technical support provide করার জন্য এই accessibility necessary। আমাদের support team শুধু official support channels-এর মাধ্যমে আপনি explicitly help request করলে আপনার applications access করবে। Specific application নিয়ে support দরকার হলে আমরা temporary passwords set করার recommendation করি।

### Third-Party Application Passwords

কিছু applications তাদের default functionality-র অংশ হিসেবে authentication credentials এবং sensitive information plaintext-এ store করতে পারে। এই behavior Appbox-এর control-এর বাইরে এবং applications' developers দ্বারা determined। আপনার Appbox environment secure, যাতে শুধু আপনি নিজের data access করতে পারেন, ফলে application-level password storage limitations থাকা সত্ত্বেও users-এর মধ্যে isolation থাকে।

## Password Security Recommendations

এই password best practices follow করে আপনি আপনার account security শক্তিশালী করতে সাহায্য করতে পারেন:

### Use Unique Passwords

* Different websites বা services জুড়ে passwords কখনও reuse করবেন না
* Multiple accounts-এ একই password ব্যবহার করা significant security risks তৈরি করে - একটি service compromised হলে আপনার সব accounts vulnerable হয়
* আপনার প্রতিটি Appbox service-এর জন্য আলাদা password তৈরি করুন

### Create Strong, Memorable Passwords

* অন্তত 12 characters long passwords লক্ষ্য করুন
* ব্যবহার করার কথা ভাবুন:
  * Movie বা book থেকে memorable quote
  * Unique phrase বা expression
  * Special characters সহ unrelated words-এর combination
* এড়িয়ে চলুন:
  * Personal information (birthdays, names, initials)
  * Common words বা phrases
  * Simple keyboard patterns (qwerty, 12345)
  * Reversed words বা simple substitutions

### Enable Two-Factor Authentication

আমরা আপনার Appbox account-এর জন্য two-factor authentication (2FA) enable করার দৃঢ় পরামর্শ দিই:

* 2FA আপনার password-এর বাইরে essential second layer of security যোগ করে
* আপনার password compromised হলেও, attackers-দের আপনার account access করতে second factor দরকার হবে
* আপনি আপনার [account security page](https://billing.appbox.co/user/security)-এ 2FA enable করতে পারেন
* Appbox authentication apps থেকে time-based tokens support করে, যার মধ্যে আছে:
  * Google Authenticator
  * Authy
  * Microsoft Authenticator 
  * Duo Mobile
  * এবং বেশিরভাগ অন্যান্য standard TOTP-compatible applications

### Consider Using a Password Manager

সব accounts-এর জন্য unique passwords তৈরি এবং মনে রাখতে অসুবিধা হলে, reputable password manager ব্যবহার করার কথা ভাবুন। এই tools strong passwords generate করতে এবং securely store করতে পারে। কিছু reliable options:

* [Bitwarden](https://bitwarden.com/)
* [LastPass](https://www.lastpass.com/)
* [LessPass](https://lesspass.com/)
* [KeePass](https://keepass.info/)

## Weak Password Policy Statement

Appbox strong passwords ব্যবহারে উৎসাহিত করলেও, ultimately আপনার account credentials-এর security আপনার responsibility। **Weak passwords বা multiple sites জুড়ে password reuse-এর কারণে আপনার account compromised হলে Appbox responsible held করা যাবে না।** Attackers সাধারণত weak passwords থাকা accounts compromise করতে brute force বা dictionary attacks ব্যবহার করে। আপনার account এবং data protect করতে উপরে বর্ণিত security guidelines follow করার আমরা দৃঢ় পরামর্শ দিই।
