Di Appbox, kami menangani keamanan password dengan serius dan menerapkan beberapa lapisan perlindungan untuk kredensial Anda. Kebijakan ini menjelaskan bagaimana kami menangani password dan memberikan panduan untuk membantu Anda membuat password yang kuat dan aman untuk layanan Appbox Anda.

## Penyimpanan Password di Appbox

Appbox memanfaatkan platform keamanan standar industri untuk memberikan pengalaman pengguna yang mulus. Bagian berikut merinci cara kami mengelola password di berbagai sistem kami.

### Client Area (WHMCS)

Appbox Client Area, yang didukung WHMCS, menangani manajemen akun, provisioning layanan, komunikasi, dan pembayaran. Password Anda di sistem ini di-hash secara aman menggunakan Bcrypt, algoritma kriptografi modern yang dirancang khusus untuk keamanan password.

### Appbox Control Panel

Control Panel kami menyediakan antarmuka terpusat untuk mengelola layanan Appbox, aplikasi terinstal, dan pengaturan akun Anda. Panel ini dibangun menggunakan framework aman dengan standar autentikasi modern.

### Password Login Control Panel

Password yang Anda gunakan untuk mengakses Appbox Control Panel dilindungi menggunakan bcrypt, fungsi hash adaptif berbasis cipher blok simetris Blowfish. Implementasi kami memanfaatkan komponen keamanan Phalcon yang menggunakan fungsi `password_hash` PHP dengan work factor kuat untuk memastikan keamanan optimal. Ini berarti:

1. Setiap hash password menyertakan salt unik untuk mencegah serangan rainbow table
2. Algoritma hashing sengaja dibuat lambat untuk memitigasi percobaan brute force
3. Work factor (cost) diatur agar serangan menjadi mahal secara komputasi
4. Sistem kami dapat upgrade ke algoritma yang lebih baru seperti Argon2i bila diperlukan untuk keamanan yang ditingkatkan

Pendekatan standar industri ini mencegah tereksposnya password bahkan jika database dikompromikan.

### Password Aplikasi

Password untuk aplikasi yang diinstal melalui Control Panel kami disimpan dalam plaintext dan dapat diakses oleh Anda maupun tim dukungan Appbox. Aksesibilitas ini diperlukan untuk menyediakan dukungan teknis saat Anda meminta bantuan. Tim dukungan kami hanya akan mengakses aplikasi Anda saat Anda secara eksplisit meminta bantuan melalui kanal dukungan resmi kami. Kami merekomendasikan pengaturan password sementara jika Anda membutuhkan dukungan untuk aplikasi tertentu.

### Password Aplikasi Pihak Ketiga

Beberapa aplikasi mungkin menyimpan kredensial autentikasi dan informasi sensitif dalam plaintext sebagai bagian dari fungsi default mereka. Perilaku ini berada di luar kendali Appbox dan ditentukan oleh developer aplikasi. Environment Appbox Anda diamankan sehingga hanya Anda yang dapat mengakses data Anda sendiri, memberikan isolasi antar pengguna meskipun ada keterbatasan penyimpanan password di level aplikasi.

## Rekomendasi Keamanan Password

Anda dapat membantu memperkuat keamanan akun dengan mengikuti praktik terbaik password berikut:

### Gunakan Password Unik

* Jangan pernah menggunakan ulang password di website atau layanan berbeda
* Menggunakan password yang sama untuk beberapa akun menciptakan risiko keamanan signifikan - jika satu layanan dikompromikan, semua akun Anda menjadi rentan
* Buat password berbeda untuk setiap layanan Appbox Anda

### Buat Password yang Kuat dan Mudah Diingat

* Usahakan password minimal 12 karakter
* Pertimbangkan menggunakan:
  * Kutipan yang mudah diingat dari film atau buku
  * Frasa atau ungkapan unik
  * Kombinasi kata yang tidak berkaitan dengan karakter khusus
* Hindari menggunakan:
  * Informasi pribadi (tanggal lahir, nama, inisial)
  * Kata atau frasa umum
  * Pola keyboard sederhana (qwerty, 12345)
  * Kata terbalik atau substitusi sederhana

### Aktifkan Autentikasi Dua Faktor

Kami sangat menyarankan mengaktifkan autentikasi dua faktor (2FA) untuk akun Appbox Anda:

* 2FA menambahkan lapisan keamanan kedua yang penting di luar password Anda
* Bahkan jika password Anda dikompromikan, penyerang membutuhkan faktor kedua untuk mengakses akun Anda
* Anda dapat mengaktifkan 2FA di [halaman keamanan akun](https://billing.appbox.co/user/security) Anda
* Appbox mendukung token berbasis waktu dari aplikasi autentikasi termasuk:
  * Google Authenticator
  * Authy
  * Microsoft Authenticator 
  * Duo Mobile
  * Dan sebagian besar aplikasi standar lain yang kompatibel dengan TOTP

### Pertimbangkan Menggunakan Password Manager

Jika Anda kesulitan membuat dan mengingat password unik untuk semua akun, pertimbangkan menggunakan password manager tepercaya. Alat ini dapat membuat password kuat dan menyimpannya secara aman untuk Anda. Beberapa opsi andal termasuk:

* [Bitwarden](https://bitwarden.com/)
* [LastPass](https://www.lastpass.com/)
* [LessPass](https://lesspass.com/)
* [KeePass](https://keepass.info/)

## Pernyataan Kebijakan Password Lemah

Meskipun Appbox mendorong penggunaan password kuat, pada akhirnya keamanan kredensial akun Anda adalah tanggung jawab Anda. **Appbox tidak dapat dimintai pertanggungjawaban jika akun Anda dikompromikan karena password lemah atau penggunaan ulang password di beberapa situs.** Penyerang umum menggunakan brute force atau dictionary attack untuk mengompromikan akun dengan password lemah. Kami sangat menyarankan mengikuti panduan keamanan yang diuraikan di atas untuk melindungi akun dan data Anda.
