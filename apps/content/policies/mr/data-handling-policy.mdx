Appbox मध्ये, आमचे records maintain करण्यासाठी आणि आमच्या services deliver करण्यासाठी आवश्यक असलेली minimum informationच आम्ही collect आणि store करतो. Appbox team तुमचा data अत्यंत privacy आणि security लक्षात घेऊन handle करण्यासाठी trained आहे आणि strong credentials व multi-factor authentication वापरण्यासह strict protocols follow करते. सर्व payment processing पूर्णपणे आमचे payment processors handle करतात, आणि payment records व त्यांच्या respective transaction IDs व्यतिरिक्त तुमची कोणतीही financial information Appbox store करत नाही.

## Payment Processing

## Logs and Analytics

### Server Metrics

- **Uses**
  - Servers ची health आणि resource usage monitor करण्यासाठी used
- **Collection**
  - User च्या slot deployment नंतर generated
- **Storage**
  - Appbox Servers = Cleartext
  - Appbox API = Cleartext
  - Posthog = Cleartext
- **Access and Reasons for Access**
  - User
    - तुमच्या slot वर installed असलेल्या विविध Linux utilities वापरून server metrics पाहता येतात. खालील major metrics पाहता येतात:
      - CPU Usage
      - Swap Usage
      - RAM Usage
      - 1/5/15 Load Average
      - तुमचे running processes
      - त्याच server वर इतर users द्वारे run केलेले processes तुम्हाला accessible नसतात
      - Input/Output metrics
      - Disk space
      - Quota disk space (तुमची allocated disk space)
      - Physical disk space
  - Appbox Team
    - वरील metrics व्यतिरिक्त, Appbox team सर्व servers मधील metrics पटकन aggregate करण्यासाठी आणि errors बद्दल team ला alert करण्यासाठी Posthog वापरते
    - Server metrics support inquiries आणि resource abuse investigate करण्यासाठी used होतात
- **Retention**
  - Indefinite

### Authentication Logs

#### Appbox Control Panel Login Attempts

- **Uses**
  - कोणतेही invalid access attempts Appbox API मध्ये logged होतात, जे specified period साठी attempts च्या certain number नंतर user ला Control Panel मध्ये login होण्यापासून restrict करते. यात समाविष्ट:
    - Username, user ने input केलेले
    - IP address
    - Number of attempts
    - Last date of attempt
- **Collection**
  - User credentials incorrectly enter करतो तेव्हा collected
- **Storage**
  - Appbox API = Cleartext
- **Access and Reasons for Access**
  - Appbox Team
    - Brute-force attempts identify करण्यासाठी used
    - Ticket system द्वारे request आणि verification नंतर users त्यांचा timeout lift आणि access attempt remove करण्याची request करू शकतात
- **Retention**
  - 24 hours किंवा user request वर

#### SSH Login Logs

- **Uses**
  - Ubuntu किंवा Debian सारख्या user-installed applications वर SSH login attempts आणि successful logins record करते
- **Collection**
  - Login attempts झाल्यावर SSH server द्वारे automatically generated
- **Storage**
  - Application container = Cleartext (typically `/var/log/auth.log` किंवा similar मध्ये)
- **Access and Reasons for Access**
  - User
    - हे logs users कडे root (sudo) access असलेल्या applications मध्ये stored असल्यामुळे, users कडे या logs वर full control आणि access असतो
    - Users गरजेनुसार हे logs view, modify, किंवा delete करू शकतात
  - Appbox team
    - Support troubleshooting दरम्यान फक्त user permission सह access available
- **Retention**
  - User द्वारे managed, कारण container वर त्यांना root access असतो
  - User ने modify केले नाही तर default log rotation policies लागू

### User's bash History

- **Uses**
  - Slot च्या shell वर user ने entered केलेले सर्व commands record करते, `$HOME/.bash_history` मध्ये located
- **Collection**
  - Shell वर entered केलेला प्रत्येक command recorded
- **Storage**
  - Appbox Servers = Cleartext
- **Access and Reasons for Access**
  - User
    - bash history file user च्या home directory मध्ये stored असल्यामुळे, users कडे या file वर full control असतो
    - Users गरजेनुसार ही file view, modify, clear, किंवा delete करू शकतात (`history -c` सारखे commands वापरून किंवा file edit/remove करून)
  - Appbox team
    - User ने entered केलेले commands correct आहेत का verify करण्यासारख्या general किंवा application-specific inquiries साठी used, पण फक्त user permission सह
    - Possible intrusions किंवा server abuse तपासण्यासाठी used, पण security investigations साठी necessary असल्यासच
- **Retention**
  - User द्वारे managed, जो history कधीही clear करू शकतो
  - By default, bash history user remove करेपर्यंत indefinitely retained

## UK Online Safety Act Compliance

UK Online Safety Act 2023 नुसार, users protect करण्यासाठी आणि regulatory requirements comply करण्यासाठी Appbox specific data handling practices implement करते:

### Content Safety Monitoring

- **Uses**
  - Illegal आणि harmful content साठी risk assessment आणि mitigation
  - Reported content चे moderation
  - Illegal content च्या distribution चे prevention
- **Collection**
  - Content reported किंवा flagged झाल्यावर generated
  - Known illegal material साठी content hash values
- **Storage**
  - Appbox API = Encrypted
  - Reports database = Cleartext
- **Access and Reasons for Access**
  - Appbox Compliance Team
    - Reported content review करण्यासाठी आणि appropriate action घेण्यासाठी
    - Legally required असल्यास UK authorities सोबत coordinate करण्यासाठी
  - UK regulatory authorities (law ने required असल्यास)
    - Online Safety Act compliance monitor करण्यासाठी
    - Serious illegal content investigate करण्यासाठी
- **Retention**
  - Report data: Resolution नंतर 2 years
  - Content hash values: Indefinite (redistribution prevention साठी)

### Risk Assessment Data

- **Uses**
  - Illegal content आणि children साठी harmful content साठी required risk assessments conduct करणे
  - Proportionate safety measures implement करणे
  - Regulatory authorities समोर compliance demonstrate करणे
- **Collection**
  - Regular platform risk assessments मार्फत generated
  - Potential risk patterns identify करण्यासाठी aggregated usage data
- **Storage**
  - Appbox Compliance Systems = Protected
- **Access and Reasons for Access**
  - Appbox Compliance Team
    - Risk assessments conduct आणि update करण्यासाठी
    - Safety measures implement करण्यासाठी
  - UK regulatory authorities (law ने required असल्यास)
    - Risk assessment duties compliance verify करण्यासाठी
- **Retention**
  - 3 years (ongoing compliance demonstrate करण्यासाठी)

### Content Reporting System

- **Uses**
  - Users ना illegal किंवा harmful content report करू देते
  - Reports चा status आणि घेतलेल्या actions track करते
- **Collection**
  - Reporting users द्वारे provided report details
  - Reported content बद्दल information
  - Reports वर घेतलेल्या actions
- **Storage**
  - Appbox API = Encrypted
- **Access and Reasons for Access**
  - Reporting user (फक्त त्यांच्या स्वतःच्या reports पर्यंत limited)
  - ज्याचा content reported झाला तो user (limited information)
  - Appbox Compliance Team
    - Reports investigate आणि resolve करण्यासाठी
  - UK regulatory authorities (law ने required असल्यास)
    - Serious illegal content reports investigate करण्यासाठी
- **Retention**
  - Resolution नंतर 2 years
